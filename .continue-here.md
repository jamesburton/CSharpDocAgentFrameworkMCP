<decisions_made>

- Chose HTTP as default transport (not stdio) because HTTP supports multi-client and remote scenarios; CLI hosts explicitly opt into stdio with --stdio
- Port 11877 chosen as default because 5000 and 8080 are commonly used by other services
- In stdio mode, Kestrel is fully disabled via `builder.WebHost.UseUrls()` with no URLs — avoids any port conflict
- Health check endpoint and MapMcp() only mapped in HTTP mode
</decisions_made>

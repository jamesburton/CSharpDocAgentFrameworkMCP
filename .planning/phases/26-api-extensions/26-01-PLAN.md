    **Step 1 — Add offset/limit parameters to GetReferences method.**

    Add two new parameters to the `GetReferences` method signature, AFTER the existing `format` parameter but BEFORE `CancellationToken`:

    ```csharp
    [Description("Result offset for pagination (default: return all)")] int offset = 0,
    [Description("Result limit; 0 = return all (default), max 200")] int limit = 0,
    ```

    The key backward-compatibility behavior: when `limit == 0` (the default), return ALL edges with no truncation. This means existing callers that pass no offset/limit get the exact same response as before.

    **Step 2 — Apply pagination to the get_references response.**

    After the `edges` list is fully collected (line ~328) and project names are resolved, apply pagination:

    ```csharp
    var totalCount = edges.Count;
    var paginatedEdges = limit > 0
        ? edges.Skip(offset).Take(Math.Min(limit, 200)).ToList()
        : edges;
    ```

    Update the response payload to use `paginatedEdges` for the `references` array, but report the full count:

    ```csharp
    var payload = new
    {
        promptInjectionWarning = hasInjectionWarning,
        total = paginatedEdges.Count,       // count of returned items (backward compat)
        totalCount = totalCount,             // total available edges
        offset = offset,
        limit = limit > 0 ? Math.Min(limit, 200) : totalCount,
        references = paginatedEdges.Select(e => new
        {
            fromId = e.From.Value,
            toId = e.To.Value,
            edgeKind = e.Kind.ToString(),
            scope = e.Scope.ToString(),
            fromProject = nodeProjectCache.GetValueOrDefault(e.From),
            toProject = nodeProjectCache.GetValueOrDefault(e.To),
        }).ToList()
    };
    ```

    CRITICAL: The `total` field must still equal the number of items in the `references` array (matching current behavior). The new `totalCount` field reports the full count. When `limit == 0`, `total == totalCount` (identical to current behavior). The `offset` and `limit` fields are additive and non-breaking.

    Update `activity?.SetTag("tool.result_count", paginatedEdges.Count);` to use `paginatedEdges`.

    Also update the TRON and markdown renderers to receive `paginatedEdges` instead of `edges`.

    **Step 3 — Create find_implementations tool.**

    Add a new tool method in DocTools after the get_references method. This tool uses IKnowledgeQueryService.GetReferencesAsync to find edges, then filters for Implements/Inherits and excludes stubs.

    ```csharp
    // ─────────────────────────────────────────────────────────────────
    // Tool: find_implementations
    // ─────────────────────────────────────────────────────────────────

    [McpServerTool(Name = "find_implementations"), Description("Find all types implementing a given interface or deriving from a base class.")]
    public async Task<string> FindImplementations(
        [Description("Stable SymbolId of the interface or base class")] string symbolId,
        [Description("Output format: json|markdown|tron")] string format = "json",
        CancellationToken cancellationToken = default)
    ```

    Implementation logic:
    1. Start activity for telemetry (`tool.find_implementations`).
    2. Validate `symbolId` is not empty — return `ErrorResponse(QueryErrorKind.InvalidInput, ...)` if blank.
    3. Create `var id = new SymbolId(symbolId);`
    4. Collect ALL references for this symbol via `_query.GetReferencesAsync(id, crossProjectOnly: false, cancellationToken)`. Buffer into a list.
    5. Catch `SymbolNotFoundException` and return `ErrorResponse(QueryErrorKind.NotFound, ...)`.
    6. Filter to edges where `edge.Kind == SymbolEdgeKind.Implements || edge.Kind == SymbolEdgeKind.Inherits` AND the edge points TO this symbol (i.e., `edge.To == id`). The `From` side is the implementing/deriving type.
    7. For each matching edge, look up the implementing node via `_query.GetSymbolAsync(edge.From, ct: cancellationToken)`. If the result is successful AND the node's `NodeKind != NodeKind.Stub`, include it.
    8. Build the response:

    ```csharp
    var payload = new
    {
        symbolId = symbolId,
        totalCount = implementations.Count,
        implementations = implementations.Select(n => new
        {
            id = n.Id.Value,
            displayName = n.DisplayName,
            kind = n.Kind.ToString(),
            fullyQualifiedName = n.FullyQualifiedName,
            projectOrigin = n.ProjectOrigin,
        }).ToList()
    };
    ```

    9. Return via `FormatResponse(format, jsonFactory, tronFactory, markdownFactory)`.
    10. For TRON format, create a simple serialization (similar to search results).
    11. For markdown, render a heading "# Implementations" with a bullet list of implementing types.

    **Step 4 — PathAllowlist security.**

    The `find_implementations` tool follows the DocTools pattern — it uses `IKnowledgeQueryService` and does not expose source spans by default. No PathAllowlist check needed on the tool itself (same as `get_references` and `search_symbols` which only check allowlist when `includeSourceSpans=true`). The security boundary is already enforced at the data access layer.

    **Step 2 — Add get_references pagination tests.**

    ```csharp
    [Fact]
    public async Task GetReferences_WithoutPagination_ReturnsAllEdges()
    {
        // Verifies backward compatibility: no offset/limit returns all edges
        var tools = CreateTools();
        var json = await tools.GetReferences(KnownId.Value);
        var root = Parse(json);

        // total == totalCount when no pagination
        root.GetProperty("total").GetInt32().Should().Be(root.GetProperty("totalCount").GetInt32());
        root.GetProperty("references").GetArrayLength().Should().Be(root.GetProperty("total").GetInt32());
    }

    [Fact]
    public async Task GetReferences_WithPagination_ReturnsSubset()
    {
        var tools = CreateTools();
        var json = await tools.GetReferences(KnownId.Value, offset: 0, limit: 2);
        var root = Parse(json);

        root.GetProperty("total").GetInt32().Should().Be(2);
        root.GetProperty("totalCount").GetInt32().Should().Be(5);  // all edges
        root.GetProperty("references").GetArrayLength().Should().Be(2);
        root.GetProperty("offset").GetInt32().Should().Be(0);
    }

    [Fact]
    public async Task GetReferences_WithOffset_SkipsEdges()
    {
        var tools = CreateTools();
        var json = await tools.GetReferences(KnownId.Value, offset: 3, limit: 10);
        var root = Parse(json);

        root.GetProperty("total").GetInt32().Should().Be(2);  // 5 total - skip 3 = 2 remaining
        root.GetProperty("totalCount").GetInt32().Should().Be(5);
    }
    ```

    Note: The GetReferences method needs `offset` and `limit` parameters added. Currently the method signature has positional params, so add them after `format`:
    `tools.GetReferences(KnownId.Value, offset: 0, limit: 2)` — use named parameters to avoid ambiguity with existing bool params.

    **Step 3 — Add find_implementations tests.**

    ```csharp
    [Fact]
    public async Task FindImplementations_ValidInterface_ReturnsImplementingTypes()
    {
        var tools = CreateTools();
        var json = await tools.FindImplementations(KnownId.Value);
        var root = Parse(json);

        root.GetProperty("totalCount").GetInt32().Should().Be(2);  // ImplClass + DerivedClass (stub excluded)
        var impls = root.GetProperty("implementations");
        impls.GetArrayLength().Should().Be(2);
    }

    [Fact]
    public async Task FindImplementations_ExcludesStubNodes()
    {
        // The stub (External::StubImpl) should not appear in results
        var tools = CreateTools();
        var json = await tools.FindImplementations(KnownId.Value);
        var root = Parse(json);

        var ids = root.GetProperty("implementations")
            .EnumerateArray()
            .Select(e => e.GetProperty("id").GetString())
            .ToList();

        ids.Should().NotContain("External::StubImpl");
    }

    [Fact]
    public async Task FindImplementations_NoImplementations_ReturnsEmpty()
    {
        var tools = CreateTools();
        // Use a symbol that exists but has no implementations
        var json = await tools.FindImplementations("MyAssembly::MyNamespace.MyClass.Run");
        var root = Parse(json);

        // May return NotFound if symbol not in stub — that's acceptable
        // OR returns empty implementations array
        if (root.TryGetProperty("implementations", out var impls))
            impls.GetArrayLength().Should().Be(0);
    }

    [Fact]
    public async Task FindImplementations_EmptyId_ReturnsError()
    {
        var tools = CreateTools();
        var json = await tools.FindImplementations("");
        var root = Parse(json);

        root.GetProperty("error").GetString().Should().Be("invalid_input");
    }
    ```

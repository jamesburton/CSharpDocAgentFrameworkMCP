2. Add new method mirroring IngestProject pattern exactly:

```csharp
[McpServerTool(Name = "ingest_solution")]
[Description("Ingest an entire .NET solution (.sln), building a queryable symbol graph across all C# projects.")]
public async Task<string> IngestSolution(
    ModelContextProtocol.Server.McpServer mcpServer,
    RequestContext<CallToolRequestParams> requestContext,
    [Description("Absolute path to .sln file")] string path,
    CancellationToken cancellationToken = default)
```

Body follows IngestProject pattern:
- Validate path not null/empty → ErrorJson("invalid_input", ...)
- `Path.GetFullPath(path)` → `_allowlist.IsAllowed(absolutePath)` → ErrorJson("access_denied", "Path is not in the configured allow list.") — EXACT same message as IngestProject
- Extract progress token from `requestContext.Params?.Meta?["progressToken"]`
- Build progressCallback (same pattern as IngestProject)
- Call `_solutionIngestionService.IngestAsync(absolutePath, progressCallback, cancellationToken)`
- Serialize result to JSON with camelCase: snapshotId, solutionName, totalProjectCount, ingestedProjectCount, totalNodeCount, totalEdgeCount, durationMs, projects (array of {name, filePath, status, reason, nodeCount, chosenTfm}), warnings
- Catch OperationCanceledException (rethrow), catch Exception → ErrorJson("ingestion_failed", ex.Message)
- Add telemetry activity like IngestProject: `DocAgentTelemetry.Source.StartActivity("tool.ingest_solution")`

**ServiceCollectionExtensions.cs changes:**

Add after the existing `IIngestionService` registration:

# Plan 06-04: Integration Gap Fixes

**Phase:** 06 — Analysis + Hosting + Gap Closure
**Closes:** MISSING-01, MISSING-02, ISearchIndex downcast tech debt

## Context

The v1.0 milestone audit identified 2 integration gaps and 1 tech debt item:

1. **MISSING-01** (medium): `DocAgent.AppHost/Program.cs` line 6 sets env var `DOCAGENT_ALLOWLIST_PATHS` but `PathAllowlist.cs` line 26 reads `DOCAGENT_ALLOWED_PATHS` — allowlist silently ignored under Aspire
2. **MISSING-02** (low): `IngestionService.IngestAsync` accepts `forceReindex` parameter but never branches on it — flag is a no-op
3. **Tech debt**: `Program.cs` line 80 downcasts `(BM25SearchIndex)app.Services.GetRequiredService<ISearchIndex>()` — `IndexAsync` is on the interface, downcast unnecessary

## Tasks

<task id="1" name="Fix AppHost env var name (MISSING-01)">

**File:** `src/DocAgent.AppHost/Program.cs`

Change `DOCAGENT_ALLOWLIST_PATHS` to `DOCAGENT_ALLOWED_PATHS` on line 6 (the `.WithEnvironment()` call).

This makes the AppHost env var match what `PathAllowlist.cs` reads at line 26.

**Verification:** Grep both files to confirm the env var name matches.

</task>

<task id="2" name="Implement forceReindex branch (MISSING-02)">

**File:** `src/DocAgent.McpServer/Ingestion/IngestionService.cs`

In `IngestAsync`, after the snapshot is built and stored (Stage 3 complete), add a conditional before the index freshness check:

```csharp
// Stage 4/4 — Index
if (reportProgress is not null)
    await reportProgress(4, 4, "Building search index...").ConfigureAwait(false);

if (forceReindex)
{
    // Force rebuild: skip freshness check, index directly from snapshot
    await _searchIndex.IndexAsync(snapshot, ct).ConfigureAwait(false);
}
else
{
    // Normal path: existing freshness-check logic
    // (keep existing code unchanged)
}
```

The key change: when `forceReindex` is true, call `IndexAsync` unconditionally, bypassing the BM25 freshness/staleness check.

**Test:** Add a test in `tests/DocAgent.Tests/Ingestion/IngestionServiceTests.cs` that verifies `forceReindex=true` triggers index rebuild even when the index is fresh.

</task>

<task id="3" name="Remove ISearchIndex downcast">

**File:** `src/DocAgent.McpServer/Program.cs`

Change line 80 from:
```csharp
var index = (BM25SearchIndex)app.Services.GetRequiredService<ISearchIndex>();
```

To:
```csharp
var index = app.Services.GetRequiredService<ISearchIndex>();
```

`IndexAsync` is defined on `ISearchIndex` (see `Abstractions.cs:32`), so no downcast is needed. This removes tight coupling to the concrete `BM25SearchIndex` type.

Also remove any `using` for `DocAgent.Indexing` in Program.cs if it becomes unused after this change.

**Verification:** `dotnet build src/DocAgent.McpServer` compiles without warnings.

</task>

Run `dotnet test` to verify all 175+ tests still pass after the three changes.

</task>

   - Diagnostic ID: `DOCAGENT001`
   - Title: "Public member lacks XML documentation"
   - Message: "'{0}' is public but has no <summary> documentation"
   - Category: "Documentation", Default severity: Warning, Enabled by default
   - `Initialize`: ConfigureGeneratedCodeAnalysis(None), EnableConcurrentExecution()
   - RegisterSymbolAction on NamedType, Method, Property, Event, Field
   - Filter: only Public, Protected, or ProtectedOrInternal accessibility

   - Check `symbol.GetDocumentationCommentXml()` for non-null/non-empty containing `<summary>`
   - Skip implicit members (constructors, operators synthesized by compiler): check `symbol.IsImplicitlyDeclared`
   - Report diagnostic at `symbol.Locations[0]`

6. Create `SuspiciousEdit/SuspiciousEditAnalyzer.cs` (ANLY-02):
   - Diagnostic ID: `DOCAGENT002`
   - Title: "Semantic change without documentation update"
   - Message: "'{0}' has a semantic change indicator but XML documentation appears unchanged"
   - Category: "Documentation", Default severity: Warning, Enabled by default
   - This analyzer detects "suspicious" patterns in current code that suggest a semantic change was made without updating docs. It uses heuristic checks on the CURRENT symbol state — it does NOT do cross-compilation diffing (that's a V2 concern).
   - Strategy: RegisterSymbolAction on Method, Property. For each public method/property:
     a. Check if the symbol has any `[Obsolete]` attribute BUT still has a doc comment that doesn't mention "obsolete" or "deprecated" → report
     b. For methods: check if any parameter has a `[NotNull]`/`[MaybeNull]`/`[AllowNull]` nullability attribute but the doc `<param>` doesn't mention null/nullable → report
     c. For methods with `<exception>` docs: check if there are `<exception>` tags referencing exception types not actually thrown (using syntax tree walk for throw statements) → report
   - These are heuristic checks per CONTEXT.md: "detect suspicious edits (semantic changes without doc/test updates)"
   - Skip ExcludeFromDocCoverage-attributed symbols
   - Skip generated code, skip non-public

   - Diagnostic ID: `DOCAGENT003`
   - Title: "Documentation coverage below threshold"
   - Message: "Documentation coverage is {0} ({1}% of public symbols documented, threshold is {2}%). Undocumented: {3}"
   - Category: "Documentation", Default severity: Warning, Enabled by default
   - Use RegisterCompilationStartAction:
     a. Track `totalPublic` (Interlocked.Increment) and `undocumented` (ConcurrentBag&lt;string&gt;)
     b. RegisterSymbolAction inside CompilationStart for NamedType, Method, Property, Event, Field
     c. Filter: Public/Protected/ProtectedOrInternal, skip ExcludeFromDocCoverage, skip IsImplicitlyDeclared
     d. Check GetDocumentationCommentXml() for `<summary>` — if missing, add to undocumented bag
     e. RegisterCompilationEndAction:
        - Read threshold from `endContext.Options.AnalyzerConfigOptionsProvider.GlobalOptions.TryGetValue("build_property.DocCoverageThreshold", ...)` — default 80
        - Calculate `coverage = (totalPublic - undocumented.Count) * 100.0 / totalPublic`
        - If coverage < threshold: report diagnostic at Location.None with summary + up to 20 undocumented symbol names
   - ConfigureGeneratedCodeAnalysis(None), EnableConcurrentExecution()

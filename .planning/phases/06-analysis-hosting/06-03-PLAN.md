   ```csharp
   var builder = DistributedApplication.CreateBuilder(args);

   var mcpServer = builder.AddProject<Projects.DocAgent_McpServer>("docagent-mcp")
       .WithEnvironment("DOCAGENT_ARTIFACTS_DIR",
           builder.Configuration["DocAgent:ArtifactsDir"] ?? "./artifacts")
       .WithEnvironment("DOCAGENT_ALLOWLIST_PATHS",
           builder.Configuration["DocAgent:AllowlistPaths"] ?? "")
       .WithHttpEndpoint(port: 8089, name: "health")
       .WithHttpHealthCheck("/health");

   builder.Build().Run();
   ```

     ```csharp
     var builder = WebApplication.CreateBuilder(args);
     // ... keep all existing DI registrations ...
     var app = builder.Build();
     app.MapHealthChecks("/health");
     // ... keep existing startup logic ...
     await app.RunAsync();
     ```
   - Add `builder.Services.AddHealthChecks()` in DI section
   - CRITICAL: Verify that `WebApplication.CreateBuilder` + `WithStdioServerTransport()` is compatible. The MCP stdio transport must still work — it reads stdin and writes to stdout.    - CRITICAL: `builder.Logging.AddConsole(o => o.LogToStandardErrorThreshold = LogLevel.Trace)` MUST remain — stdout purity for MCP JSON-RPC is non-negotiable. If WebApplication logs to stdout by default, ensure it's suppressed.
   - Add `Microsoft.Extensions.Diagnostics.HealthChecks` package if not already included via framework ref.

   FALLBACK: If WebApplication + stdio MCP transport proves incompatible, use `Microsoft.Extensions.Diagnostics.HealthChecks` with a simple `IHostedService` that starts an `HttpListener` on port 8089 for `/health`. This is less clean but doesn't risk MCP transport interference.

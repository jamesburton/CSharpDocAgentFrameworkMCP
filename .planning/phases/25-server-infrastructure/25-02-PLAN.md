<objective>
Add per-tool-category rate limiting via token-bucket algorithm to prevent stuck-agent retry storms from overwhelming the server.

Purpose: Protect the MCP server from runaway clients that rapidly retry tool calls. Query and ingestion tools have separate rate limits so heavy querying cannot block ingestion and vice versa. Rate-limited requests receive a structured error (not an unhandled exception) with retry guidance.
Output: ToolRateLimiter singleton, RateLimitFilter, RateLimitOptions configuration, registered in Program.cs, with unit tests.
</objective>

    **Public API:**
    ```csharp
    /// <summary>
    /// Attempts to acquire a rate limit lease for the given tool name.
    /// Returns (allowed: true, lease) if within limit, or (allowed: false, retryAfterMs) if rate-limited.
    /// </summary>
    public (bool Allowed, RateLimitLease? Lease, int RetryAfterMs) TryAcquire(string toolName)
    ```

    Implementation:
    1. If rate limiting is disabled, return `(true, null, 0)`.
    2. Determine bucket: `IsIngestionTool(toolName)` returns true for `"ingest_project"` and `"ingest_solution"`, false for everything else.
    3. Call `limiter.AttemptAcquire(permitCount: 1)` on the appropriate limiter.
    4. If `lease.IsAcquired`, return `(true, lease, 0)`.
    5. If not acquired, extract `RetryAfter` from `lease.TryGetMetadata(MetadataName.RetryAfter, out TimeSpan retryAfter)`. Convert to milliseconds (default 1000 if metadata unavailable). Dispose the lease. Return `(false, null, retryAfterMs)`.

    ```csharp
    private static bool IsIngestionTool(string toolName) =>
        toolName is "ingest_project" or "ingest_solution";
    ```

    Make `IsIngestionTool` internal for testability (InternalsVisibleTo is already set for DocAgent.Tests).

    **Dispose pattern:** Dispose both `TokenBucketRateLimiter` instances.

    Create `src/DocAgent.McpServer/Filters/RateLimitFilter.cs`:

    - Follow EXACTLY the same extension method pattern as `AuditFilter.AddAuditFilter()`.

    ```csharp
    public static IMcpServerBuilder AddRateLimitFilter(this IMcpServerBuilder builder)
    {
        return builder.WithRequestFilters(filters =>
        {
            filters.AddCallToolFilter(next => async (context, cancellationToken) =>
            {
                var limiter = context.Services!.GetRequiredService<ToolRateLimiter>();
                var toolName = context.Params?.Name ?? "unknown";

                var (allowed, lease, retryAfterMs) = limiter.TryAcquire(toolName);

                if (!allowed)
                {
                    return new CallToolResult
                    {
                        Content = [new TextContentBlock
                        {
                            Text = JsonSerializer.Serialize(new
                            {
                                error = "rate_limited",
                                message = $"Too many requests. Please retry after {retryAfterMs}ms.",
                                retryAfterMs,
                            }, s_jsonOptions),
                        }],
                        IsError = true,
                    };
                }

                try
                {
                    return await next(context, cancellationToken);
                }
                finally
                {
                    lease?.Dispose();
                }
            });
        });
    }
    ```

    Add a static `JsonSerializerOptions` field matching the AuditFilter pattern:
    ```csharp
    private static readonly JsonSerializerOptions s_jsonOptions = new()

        PropertyNamingPolicy = JsonNamingPolicy.CamelCase,
        WriteIndented = false,
    };
    ```

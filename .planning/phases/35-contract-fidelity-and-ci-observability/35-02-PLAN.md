Fix INT-02: Two test methods in `TypeScriptSidecarIntegrationTests` use early-return guards that cause tests to show as "Passed" in CI even though no assertions ran.

**Method 1: `RealSidecar_SimpleProject_Produces_Valid_Snapshot`** (~line 82):
- Change `[Fact]` to `[Fact(Skip = "Requires Node.js and compiled sidecar. Set RUN_SIDECAR_TESTS=true and run: dotnet test --filter 'FullyQualifiedName~TypeScriptSidecarIntegration'")]`
- Remove the early-return guard lines (lines 85-86): `if (Environment.GetEnvironmentVariable("RUN_SIDECAR_TESTS") != "true") return;`

**Method 2: `RealSidecar_Snapshot_Is_Queryable`** (~line 142):
- Same change: replace `[Fact]` with `[Fact(Skip = "Requires Node.js and compiled sidecar. Set RUN_SIDECAR_TESTS=true and run: dotnet test --filter 'FullyQualifiedName~TypeScriptSidecarIntegration'")]`
- Remove the early-return guard lines (lines 145-146).

This means these tests will ALWAYS show as "Skipped" in standard `dotnet test` output. This is correct and intentional per research Pattern 1 — static `[Fact(Skip=...)]` is the right choice for tests requiring external infrastructure. The skip message tells developers how to run them manually.

Do NOT use `.WaitFor()` or any conditional skip logic. The static Skip attribute is the project-standard approach.

### TypeScript Sidecar Integration Tests

`TypeScriptSidecarIntegrationTests` validates the real Node.js sidecar pipeline (no mocks). These tests are always skipped in standard `dotnet test` runs via `[Fact(Skip=...)]`.

**Prerequisites:**
- Node.js installed and on PATH
- Sidecar built: `cd src/ts-symbol-extractor && npm install && npm run build`

**Running manually:**
```bash
RUN_SIDECAR_TESTS=true dotnet test --filter "FullyQualifiedName~TypeScriptSidecarIntegration"
```

`TypeScriptIngestionBenchmarks` (in the Benchmarks project) measures sidecar ingestion performance. Requires the same prerequisites as sidecar integration tests.

**Running:**
```bash
dotnet run -c Release --project tests/DocAgent.Benchmarks
```

**CI integration:** Not included in standard `dotnet test`. To add to CI, ensure Node.js and compiled sidecar are available in the CI environment, then run the benchmark project directly.

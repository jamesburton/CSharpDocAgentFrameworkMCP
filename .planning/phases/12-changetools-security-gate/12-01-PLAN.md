**Step 2:** Add 3 new test methods — one per tool method — that verify denial behavior:

```csharp
[Fact]
public async Task ReviewChanges_PathDenied_ReturnsAccessDenied()
{
    var (hashA, hashB) = await SaveBreakingPairAsync();
    // Default DocAgentServerOptions has no AllowedPaths — only CWD allowed — _tempDir denied
    var allowlist = new PathAllowlist(Options.Create(new DocAgentServerOptions()));
    var tools = CreateTools(allowlist: allowlist);

    var json = await tools.ReviewChanges(hashA, hashB);
    var root = Parse(json);

    root.TryGetProperty("error", out var err).Should().BeTrue();
    err.GetString().Should().Be("not_found");
}
```

Create analogous tests:
- `FindBreakingChanges_PathDenied_ReturnsAccessDenied` — calls `tools.FindBreakingChanges(hashA, hashB)` with restrictive allowlist
- `ExplainChange_PathDenied_ReturnsAccessDenied` — calls `tools.ExplainChange(hashB, someSymbolId)` with restrictive allowlist (use a symbol ID from the saved snapshot; exact ID depends on what `SaveBreakingPairAsync` creates — inspect the snapshot to find a valid node ID)

**Key details:**
- Use `new PathAllowlist(Options.Create(new DocAgentServerOptions()))` for the restrictive allowlist (no AllowedPaths = deny non-CWD paths)
- The `_tempDir` used by SnapshotStore is under `Path.GetTempPath()`, which is outside CWD, so default allowlist denies it

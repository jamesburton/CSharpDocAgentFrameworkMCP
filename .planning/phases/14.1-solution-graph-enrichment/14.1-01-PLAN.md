**SolutionIngestionResult change:**
Add `SolutionSnapshot? Snapshot = null` as a new optional parameter at the end of the SolutionIngestionResult record. This is backward-compatible — all existing callers pass positional args and get null default.

**SolutionIngestionService changes — three additive capabilities:**

1. **ProjectWalkContext struct** (internal, at bottom of class):
```csharp
private readonly record struct ProjectWalkContext(
    string ProjectName,
    HashSet<string> SolutionProjectNames,
    List<SymbolNode> StubNodes,
    HashSet<string> SeenStubIds);
```

2. **Primitive filter** — static `HashSet<string>` with common types to exclude from stub creation:
System.Object, System.String, System.Boolean, System.Byte, System.SByte, System.Int16, System.UInt16, System.Int32, System.UInt32, System.Int64, System.UInt64, System.Single, System.Double, System.Decimal, System.Char, System.Void, System.IntPtr, System.UIntPtr, System.Threading.Tasks.Task, System.Threading.Tasks.Task`1, System.Collections.Generic.IEnumerable`1, System.Collections.Generic.IList`1, System.Collections.Generic.IReadOnlyList`1, System.Collections.Generic.ICollection`1, System.Collections.Generic.IDictionary`2, System.ValueType, System.Enum, System.Attribute, System.Exception, System.IDisposable, System.EventArgs.

3. **ClassifyScope helper** (private static):
Takes `ITypeSymbol targetType`, `string currentProjectName`, `HashSet<string> solutionProjectNames`. Returns `EdgeScope`:
- If `targetType.ContainingAssembly` is null → `EdgeScope.External`
- If assembly name not in `solutionProjectNames` → `EdgeScope.External`
- If assembly name != currentProjectName (OrdinalIgnoreCase) → `EdgeScope.CrossProject`
- Otherwise → `EdgeScope.IntraProject`

4. **MaybeAddStubNode helper** (private static):
Takes `ITypeSymbol externalType`, `in ProjectWalkContext ctx`. If type is primitive (check `IsPrimitive`), return. Use `type.OriginalDefinition` for consistent dedup. Get ID via `GetSymbolId(externalType.OriginalDefinition)`. If `seenStubIds.Add(id.Value)` returns false, skip. Otherwise create a `SymbolNode` with `NodeKind: NodeKind.Stub`, `ProjectOrigin: externalType.ContainingAssembly?.Name`, and add to `ctx.StubNodes`.

5. **Update WalkTypeInline and WalkNamespaceInline signatures** to accept `in ProjectWalkContext ctx`:
- `WalkNamespaceInline(ns, nodes, edges)` → `WalkNamespaceInline(ns, nodes, edges, in ctx)`
- `WalkTypeInline(type, typeId, nodes, edges)` → `WalkTypeInline(type, typeId, nodes, edges, in ctx)`
- In WalkTypeInline, when creating Inherits/Implements edges, pass `ClassifyScope(...)` as the `Scope` parameter on `new SymbolEdge(...)`.
- After creating each Inherits/Implements edge where scope is `External`, call `MaybeAddStubNode(...)`.

6. **Update IngestAsync** to build SolutionSnapshot:
- Before project loop: build `solutionProjectNames` HashSet from `chosenProjects.Select(p => GetProjectName(p))` and also `projectIdToName` Dictionary mapping `ProjectId → project name` from chosenProjects (using `Path.GetFileNameWithoutExtension(p.FilePath!)` as name, consistent with existing `projectName` logic).
- Accumulate `List<ProjectEntry>`, `List<ProjectEdge>`, `List<SymbolGraphSnapshot>` during the project loop.
- For each successfully compiled project: extract `project.ProjectReferences`, resolve each via `projectIdToName`, build `ProjectEntry(projectName, filePath, directDeps)` and `ProjectEdge(projectName, dep)` for each dep.
- Create a per-project `SymbolGraphSnapshot` from `stampedNodes` and `projectEdges` for that project, add to `perProjectSnapshots` list.
- Create `ProjectWalkContext` per project (with shared `stubNodes` and `seenStubIds` lists) and pass to `WalkCompilationAsync` → `WalkNamespaceInline`.
- After project loop: add all `stubNodes` to `allNodes`.
- **Circular reference detection:** After building all `ProjectEdge` records, run a simple DFS cycle detector. For each cycle found, log a warning via `_logger.LogWarning("Circular project reference detected: {Cycle}", cyclePath)` and add to `warnings` list. Include both edges — do not remove them.
- Construct `SolutionSnapshot` with all accumulated data.
- Pass `Snapshot: solutionSnapshot` to the `SolutionIngestionResult` constructor.

**Important constraints (from locked decisions):**
- Edge scope set at construction time — NO post-classification pass
- Only directly referenced types create edges — do NOT walk inheritance hierarchies
- Type-level stubs only — do NOT enumerate members of external types
- Use `project.FilePath` as stable key for project identity (not `project.Name` which may have TFM suffix)

7. **`IngestAsync_DetectsCircularProjectReferences`** — Pipeline override returns a result with circular A→B→A ProjectEdges. Assert both edges present in `ProjectDependencies` and a warning about circular reference exists in `Warnings`.

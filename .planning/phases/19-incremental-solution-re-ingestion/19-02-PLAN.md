**IncrementalSolutionIngestionService** — new class implementing `ISolutionIngestionService`:

Constructor deps: `SnapshotStore`, `ILogger<IncrementalSolutionIngestionService>`, `SolutionIngestionService` (for full-reingest fallback). Follow existing DI patterns.

Add `forceFullReingest` optional bool parameter to `ISolutionIngestionService.IngestAsync`. This is a LOCKED DECISION. Update the interface to:
```csharp
Task<SolutionIngestionResult> IngestAsync(
    string slnPath,
    Func<int, int, string, Task>? reportProgress,
    CancellationToken cancellationToken,
    bool forceFullReingest = false);
```
Update `SolutionIngestionService` to accept and ignore this parameter (it always does full ingest).

**IngestAsync flow** (see 19-RESEARCH.md "Full Incremental Solution Ingestion Skeleton"):

1. If `forceFullReingest` → delegate to `SolutionIngestionService.IngestAsync` immediately.
2. Open MSBuildWorkspace, load solution (always required — locked decision).
3. Deduplicate multi-targeted projects (reuse existing `SolutionIngestionService` logic — extract to shared helper or call through PipelineOverride).
4. Load previous `SolutionSnapshot` from `SnapshotStore`. Use a pointer file `{artifactsDir}/latest-{solutionName}.ptr` containing the content hash of the most recent solution snapshot. If no pointer or snapshot → full ingest.
5. Check `HasStructuralChange` — if true → full ingest + log reason.
6. Compute per-project manifests via `SolutionManifestStore.ComputeProjectManifestAsync`.
7. Load previous per-project manifests via `SolutionManifestStore.LoadAsync`.
8. Determine directly-changed projects via `FileHasher.Diff`.
9. Build project edges, call `DependencyCascade.ComputeDirtySet`.
10. Call `DependencyCascade.TopologicalSort`, iterate in order.
11. For each project:
    - If NOT dirty: log "Skipped {Project} (unchanged)", preserve nodes/edges/stubs from previous snapshot. Set `ProjectIngestionStatus.Status = "skipped"` with `Reason = "unchanged"`.
    - If dirty: re-ingest (GetCompilationAsync → walk namespace → accumulate nodes/edges). Standard existing logic from SolutionIngestionService.
12. **Stub lifecycle** (CRITICAL for INGEST-04):
    - Collect all `NodeKind.Stub` nodes from previous snapshot as baseline.
    - Re-ingested projects regenerate stubs fresh (via `MaybeAddStubNode` / `seenStubIds`).
    - After loop: union baseline stubs with newly-generated stubs, dedup by `SymbolId` (new wins).
    - Prune stubs that have zero surviving inbound edges from any `NodeKind.Real` node.
13. Sort nodes/edges via `SymbolSorter` for determinism.
14. Save manifests via `SolutionManifestStore.SaveAsync`. Clean orphaned manifests.
15. Save snapshot via `SnapshotStore`. Update pointer file.
16. Return `SolutionIngestionResult` with per-project statuses.

Add `ProjectsSkippedCount` and `ProjectsReingestedCount` computed properties to `SolutionIngestionResult` (derived from `Projects` list, no constructor change needed).

**DI registration**: In `DependencyInjection.cs`, register `IncrementalSolutionIngestionService` as the primary `ISolutionIngestionService` singleton. Keep `SolutionIngestionService` registered by concrete type so `IncrementalSolutionIngestionService` can inject it for fallback.

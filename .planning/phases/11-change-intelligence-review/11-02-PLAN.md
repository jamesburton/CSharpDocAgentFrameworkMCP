**Tool 1: review_changes**
```csharp
[McpServerTool(Name = "review_changes"), Description("Review all changes between two snapshot versions, grouped by severity with unusual pattern detection.")]
public async Task<string> ReviewChanges(
    [Description("Before snapshot version (content hash)")] string versionA,
    [Description("After snapshot version (content hash)")] string versionB,
    [Description("Include doc-only/trivial changes (default: false)")] bool verbose = false,
    [Description("Output format: json|markdown|tron")] string format = "json",
    CancellationToken cancellationToken = default)
```

Implementation:
1. OpenTelemetry: `DocAgentTelemetry.Source.StartActivity("tool.review_changes", ActivityKind.Internal)` + SetTag
2. Load snapshots via `_snapshotStore.LoadAsync(versionA/B, cancellationToken)`. If null → `ErrorResponse(QueryErrorKind.SnapshotMissing, ...)`
3. Call `SymbolGraphDiffer.Diff(snapshotA, snapshotB)` wrapped in try/catch `ArgumentException` → `ErrorResponse(QueryErrorKind.InvalidInput, ex.Message)`
4. Call `ChangeReviewer.Analyze(diff, verbose)` to get `ChangeReviewReport`
5. **Populate ImpactScope** for each finding: Look up `snapshotB.Edges.Where(e => e.To == new SymbolId(finding.SymbolId)).Select(e => e.From.Value).ToList()`. Update findings with populated ImpactScope (create new records since they're immutable).
6. Scan doc-derived text through `PromptInjectionScanner.Scan()` on Description and Remediation fields.
7. Format JSON output matching the research JSON shape. Format markdown as severity-grouped sections. Format tron via new `TronSerializer.SerializeChangeReview()`.

**Tool 2: find_breaking_changes**
```csharp
[McpServerTool(Name = "find_breaking_changes"), Description("Find public API breaking changes between two snapshots. CI-optimized minimal output.")]
public async Task<string> FindBreakingChanges(
    [Description("Before snapshot version (content hash)")] string versionA,
    [Description("After snapshot version (content hash)")] string versionB,
    [Description("Output format: json|markdown|tron")] string format = "json",
    CancellationToken cancellationToken = default)
```

Implementation:
1. Same snapshot loading and diff pattern as review_changes.
2. Filter `diff.Changes.Where(c => c.Severity == ChangeSeverity.Breaking)` — do NOT include DocComment or Informational changes.
3. Return minimal shape: `{ beforeVersion, afterVersion, breakingCount, breakingChanges: [{ symbolId, displayName, description }] }`.
4. No unusual pattern detection. No verbose parameter. CI-focused.

**Tool 3: explain_change**
```csharp
[McpServerTool(Name = "explain_change"), Description("Get a detailed human-readable explanation of changes to a specific symbol between two snapshots.")]
public async Task<string> ExplainChange(
    [Description("Before snapshot version (content hash)")] string versionA,
    [Description("After snapshot version (content hash)")] string versionB,
    [Description("SymbolId to explain")] string symbolId,
    [Description("Output format: json|markdown|tron")] string format = "json",
    CancellationToken cancellationToken = default)
```

Implementation:
1. Same snapshot loading and diff pattern.
2. Filter `diff.Changes.Where(c => c.SymbolId == new SymbolId(symbolId))`. If empty → `ErrorResponse(QueryErrorKind.NotFound, $"No changes found for symbol '{symbolId}'")`
3. For each change, build: category, severity (mapped to tier name), before/after strings from detail fields (use switch on Category — see research code example), whyItMatters text (generated based on category + severity), impactScope (from `snapshotB.Edges.Where(e => e.To == new SymbolId(symbolId) && e.Kind == SymbolEdgeKind.Calls)`), remediation text.
4. Build `whyItMatters` based on category:
   - Signature Breaking: "This {description}. Any caller will fail to compile."
   - Nullability: "Return type/parameter nullability changed. Callers without null checks will get warnings/errors."
   - Accessibility widening: "Public API surface increased. This symbol is now accessible to more consumers."
   - Constraint removal: "Generic constraints removed. Code that depended on constraint guarantees may break."
5. Scan all text through PromptInjectionScanner.
6. Return JSON matching research explain_change shape. ChangeType is "Added"/"Removed"/"Modified" from first change entry.

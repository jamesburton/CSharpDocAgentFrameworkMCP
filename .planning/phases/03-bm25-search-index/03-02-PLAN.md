<task type="auto">
  <name>Task 1: Add FSDirectory persistence and freshness check to BM25SearchIndex</name>
  <files>src/DocAgent.Indexing/BM25SearchIndex.cs</files>
  <action>
Update `BM25SearchIndex` to support persistent filesystem-based indexing tied to snapshot content hash.

**Persistence path convention:** `{artifactsDir}/{snapshot.ContentHash}.lucene/`

**IndexAsync changes:**
1. If `snapshot.ContentHash` is null or empty, throw `ArgumentException("Snapshot must have a ContentHash for persistent indexing")` — or if using injected RAMDirectory, skip the freshness check and always rebuild.
2. When using FSDirectory (not injected directory): compute index path as `Path.Combine(artifactsDir, $"{snapshot.ContentHash}.lucene")`.
3. **Freshness check:** Before rebuilding, call a private `IsIndexFresh(dir, contentHash)` method:
   - If `DirectoryReader.IndexExists(dir)` is true, open reader, check `reader.IndexCommit.UserData["snapshotHash"]` equals `snapshot.ContentHash`.
   - If fresh, populate `_nodes` dictionary from snapshot (needed for `GetAsync`) and return early — skip re-indexing.
4. If not fresh, proceed with full index build (existing logic from 03-01).
5. After `writer.Commit()`, store snapshot hash: `writer.SetCommitData(new Dictionary<string, string> { { "snapshotHash", snapshot.ContentHash } })` then commit again.

**SearchAsync changes:**
- When using FSDirectory, open the directory at the persisted path. The search logic remains the same.
- Handle case where no index directory exists yet — yield break.

**LoadIndexAsync (new public method):**
Add a method `Task LoadIndexAsync(string contentHash, SymbolGraphSnapshot snapshot, CancellationToken ct)` that:
1. Opens the FSDirectory at `{artifactsDir}/{contentHash}.lucene/`
2. Verifies freshness via `IndexCommit.UserData`
3. Populates `_nodes` from the snapshot for `GetAsync`
4. Sets internal state so `SearchAsync` uses this directory

This enables loading a persisted index without re-indexing — the primary INDX-03 use case.

**Dispose:** Ensure FSDirectory instances are properly disposed.

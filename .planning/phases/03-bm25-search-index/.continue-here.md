<decisions_made>

- Regex pattern `(?<=[a-z])(?=[A-Z])|(?<=[A-Z])(?=[A-Z][a-z])` for both lower→upper and acronym→capitalized splits
- BM25 k1=2.0, b=0.5 for symbolName field — higher term frequency saturation, reduced length normalization
- Two-commit protocol for FSDirectory persistence — first commit flushes docs, second stores snapshotHash in UserData
- Dictionary<SymbolId, SymbolNode> for GetAsync (same pattern as InMemorySearchIndex) rather than Lucene stored fields
- NoWarn NU1701 added for Lucene.Net net8.0→net10.0 compatibility

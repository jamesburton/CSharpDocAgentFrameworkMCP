    **Step 1 — Create repo-level nuget.config with package source mapping.**
    The global NuGet config has multiple sources (nuget.org + QHubPackages) causing NU1507 under CPM. Create `nuget.config` in the repo root with package source mapping that routes all packages to nuget.org:

    ```xml
    <?xml version="1.0" encoding="utf-8"?>
    <configuration>
      <packageSources>
        <clear />
        <add key="nuget.org" value="https://api.nuget.org/v3/index.json" />
      </packageSources>
      <packageSourceMapping>
        <packageSource key="nuget.org">
          <package pattern="*" />
        </packageSource>
      </packageSourceMapping>
    </configuration>
    ```

    This resolves NU1507 by establishing explicit source mapping. Using `<clear />` ensures only nuget.org is used for this repo, ignoring the global QHubPackages source that is not needed.

    Add a new PropertyGroup (or extend the existing one) with:

    ```xml

    <NuGetAuditMode>direct</NuGetAuditMode>
    ```

    Using `NuGetAuditMode=direct` per research recommendation — audits only directly-referenced packages. This avoids surfacing non-actionable transitive vulnerabilities from the Microsoft.Build.Tasks.Core chain brought in by Workspaces.MSBuild.

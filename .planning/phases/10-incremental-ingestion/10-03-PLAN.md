Create `tests/DocAgent.Tests/IncrementalIngestion/IncrementalCorrectnessTests.cs`.

These tests use real Roslyn compilation against a small synthetic .csproj written to a temp directory. Follow the pattern from existing `DeterminismTests.cs` and `RoslynSymbolGraphBuilderTests.cs` for creating temp projects.

**Test setup helper:** Create a temp directory with a minimal `.csproj` and 2-3 `.cs` files. Use `<TargetFramework>net10.0</TargetFramework>` and `<LangVersion>preview</LangVersion>` matching the project.

**Tests:**

1. `Incremental_with_no_changes_produces_identical_snapshot` — Ingest full, then ingest incremental with same files. Normalize `CreatedAt`, `ContentHash`, `IngestionMetadata` to null. Byte-compare via MessagePack serialization. Must be identical.

2. `Incremental_after_file_modification_matches_full_reingestion` — Ingest full (baseline). Modify one `.cs` file (e.g., add a method). Run incremental. Run full again. Normalize timestamps/metadata/ContentHash. Byte-compare incremental vs second full. Must be identical.

3. `Incremental_after_file_addition_matches_full_reingestion` — Ingest full. Add a new `.cs` file to the project. Run incremental. Run full. Compare.

4. `Incremental_after_file_removal_matches_full_reingestion` — Ingest full. Delete a `.cs` file. Run incremental. Run full. Compare.

**Normalization for comparison:**
```csharp
static SymbolGraphSnapshot Normalize(SymbolGraphSnapshot s) =>
    s with { CreatedAt = default, ContentHash = null, IngestionMetadata = null };
```

Then serialize both with `MessagePackSerializer.Serialize(Normalize(s), ContractlessStandardResolver.Options)` and `SequenceEqual`.

**If tests require MSBuild:** These are integration tests and may be slow (~10-30s each). Mark with `[Trait("Category", "Integration")]` so they can be filtered. Ensure the test project `.csproj` is self-contained (no ProjectReferences that require building other projects).

**Fallback:** If real Roslyn compilation in tests is too fragile (MSBuild SDK resolution issues in test context), use the `BuildOverride` seam to inject a deterministic mock builder that returns predictable nodes based on file content. Document which approach was taken.

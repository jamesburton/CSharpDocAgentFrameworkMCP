  <action>
Create `src/DocAgent.Ingestion/IncrementalIngestionEngine.cs`:

**Class:** `public sealed class IncrementalIngestionEngine`

**Constructor dependencies:**
- `RoslynSymbolGraphBuilder builder` (or `ISymbolGraphBuilder`)
- `SnapshotStore store`
- `string artifactsDir` (for manifest storage)
- `Action<string>? logWarning = null`

**Internal injectable for testing:**
- `internal Func<ProjectInventory, DocInputSet, CancellationToken, Task<SymbolGraphSnapshot>>? BuildOverride { get; set; }` — same PipelineOverride pattern as IngestionService

**Main method:** `public async Task<SymbolGraphSnapshot> IngestAsync(ProjectInventory inventory, DocInputSet docs, SymbolGraphSnapshot? previousSnapshot, bool forceFullReingestion, CancellationToken ct)`

**Algorithm:**
1. Record `startedAt = DateTimeOffset.UtcNow`, generate `runId = Guid.NewGuid().ToString("N")`
2. Hash all source files across all project directories — enumerate `.cs` files under each project's directory, call `FileHasher.ComputeManifestAsync`
3. Load previous `FileHashManifest` from `{artifactsDir}/file-hashes.json` (null if first run)
4. Compute `ManifestDiff` between previous and current manifest
5. If `forceFullReingestion` or no previous snapshot or no previous manifest: full re-ingest all projects
6. Otherwise, determine which PROJECT FILES need re-parse: for each project file in `inventory.ProjectFiles`, check if any `.cs` file under that project's directory appears in `ManifestDiff.ChangedFiles` or `ManifestDiff.AddedFiles`. If yes, that project needs re-parse.
7. If no projects need re-parse (no changes detected), return previous snapshot with updated metadata
8. Build a filtered `ProjectInventory` with only changed project files, call `builder.BuildAsync` (or `BuildOverride`) on that subset
9. **Merge:** Preserve nodes from previous snapshot whose `SourceSpan.FilePath` does NOT belong to any re-parsed project. Combine with newly built nodes. Deduplicate by `SymbolId` (keep new over old). Same for edges.
10. Sort via `SymbolSorter.SortNodes` / `SymbolSorter.SortEdges`
11. Build `IngestionMetadata` with file changes and affected symbol IDs (symbols whose source file was in the changed set)
12. Construct new `SymbolGraphSnapshot` with merged nodes/edges and metadata
13. Save new `FileHashManifest` to `{artifactsDir}/file-hashes.json` ONLY after successful snapshot construction (per Pitfall 4 from research)

**Helper:** `private static bool IsFileInProjectDirectory(string filePath, string projectFilePath)` — checks if a source file path is under the same directory as the project file.

**Tests** in `IncrementalIngestionEngineTests.cs`:
- Use `BuildOverride` to avoid real Roslyn compilation
- Test: first run (no previous) does full ingestion
- Test: no changes returns previous snapshot (with updated metadata, WasFullReingestion=false)
- Test: changed file in one project re-parses only that project, preserves other project's symbols
- Test: forceFullReingestion=true re-parses all projects
- Test: removed file's symbols not in output
- Test: IngestionMetadata correctly records file changes

Create `DeterminismTests.cs` with [Trait("Category", "Integration")] tests:

**Key requirement:** For determinism, `CreatedAt` must be fixed to a constant value (not `DateTimeOffset.UtcNow`). The RoslynSymbolGraphBuilder should accept an optional timestamp parameter, or the test should use reflection/with-expression to fix the timestamp after building.

**Tests:**

1. `FullPipeline_produces_identical_snapshots_across_runs` — The primary determinism test:
   - Create LocalProjectSource, XmlDocParser, InheritDocResolver, RoslynSymbolGraphBuilder
   - Run the full pipeline against `src/DocAgent.Core/DocAgent.Core.csproj` twice
   - Fix CreatedAt to the same constant in both snapshots
   - Serialize both with MessagePack (ContractlessStandardResolver)
   - Assert byte arrays are identical: `bytes1.SequenceEqual(bytes2)` must be true
   - Also verify content hashes match

2. `FullPipeline_content_hash_stable_across_runs` — Similar to test 1 but focused on SnapshotStore:
   - Run pipeline twice, save both via SnapshotStore
   - Assert ContentHash values are identical
   - Assert the .msgpack files are byte-identical

3. `FullPipeline_deterministic_with_multiple_projects` — Run against `src/DocAgentFramework.sln` (which has multiple projects). Fix timestamps. Assert byte-identical output.
   - This catches ordering issues across multiple projects, partial types spanning files, etc.

4. `Nodes_sorted_by_ordinal_id` — After building, verify `snapshot.Nodes` are sorted by `Id.Value` using `StringComparer.Ordinal`.

5. `Edges_sorted_deterministically` — After building, verify `snapshot.Edges` are sorted by (From, To, Kind).

**Implementation notes:**
- These tests are slow (each run opens MSBuildWorkspace). Mark with [Trait("Category", "Integration")].
- If RoslynSymbolGraphBuilder doesn't accept a timestamp parameter, either add one (preferred) or use `snapshot with { CreatedAt = fixedTimestamp }` after building.
- Use `MessagePackSerializer.Serialize(snapshot, ContractlessStandardResolver.Options)` for byte comparison.
- If any non-determinism is found during test writing, fix the source (likely a missing sort or Dictionary iteration).

**Replace the stub XmlDocParser** with a full implementation. Change the interface to work with raw XML strings per-symbol (not per-assembly), since the RoslynSymbolGraphBuilder (Plan 03) will call `ISymbol.GetDocumentationCommentXml()` per symbol and pass the XML here.

**New XmlDocParser API:**
```csharp
public sealed class XmlDocParser
{
    /// Parse a single symbol's XML doc comment into a DocComment.
    /// xmlContent is the raw XML from ISymbol.GetDocumentationCommentXml().
    /// Returns null if xmlContent is null/empty (caller handles placeholder synthesis).
    public DocComment? Parse(string? xmlContent);
}
```

Remove the old `IXmlDocParser` interface and `DocInputSet`-based API — the graph builder will call `Parse` per-symbol directly.

**Parsing logic using System.Xml.Linq.XDocument:**
1. Wrap xmlContent in try/catch for XmlException.
2. Parse with `XDocument.Parse(xmlContent)`.
3. Find the `<member>` element (or root if no member wrapper).
4. Extract each element into DocComment fields:
   - `<summary>` → `Summary` (inner text, trim whitespace)
   - `<remarks>` → `Remarks`
   - `<param name="x">` → `Params["x"]`
   - `<typeparam name="T">` → `TypeParams["T"]`
   - `<returns>` → `Returns`
   - `<example>` → add to `Examples` list
   - `<exception cref="T:System.ArgumentException">` → add `(Type, Description)` to `Exceptions`
   - `<seealso cref="...">` → add cref value to `SeeAlso`
5. On XmlException (malformed XML): attempt recovery by wrapping in `<doc>...</doc>`. If still failing, create DocComment with raw text in Summary and set a warning. Add an `IsWarning` or `HasParseWarning` boolean property — use a wrapper or just note it in the Summary prefix like "[Parse warning] ...".

**Per user decision:** Best-effort parse for malformed XML — extract what's parseable, don't fail the whole ingestion.

**InheritDocResolver:**
```csharp
public sealed class InheritDocResolver
{
    /// Resolve inheritdoc for a symbol by walking its override/interface chain.
    /// symbolDocXml: the raw XML that may contain <inheritdoc/>.
    /// getBaseDocXml: callback to get XML for a base symbol by doc comment ID.
    /// Returns resolved DocComment or null.
    public DocComment? Resolve(
        string? symbolDocXml,
        Func<string, string?> getBaseDocXml,
        XmlDocParser parser,
        int maxDepth = 10);
}
```

Logic:
1. If symbolDocXml is null/empty, return null (caller synthesizes placeholder).
2. Parse symbolDocXml. If it contains `<inheritdoc/>` or `<inheritdoc cref="..."/>`:
   - If `cref` specified: call `getBaseDocXml(cref)` to get the target's XML.
   - If no `cref`: caller provides the overridden/interface member's doc ID via the callback.
3. Track visited doc IDs in a `HashSet<string>` to detect cycles.
4. Recurse up to `maxDepth` levels.
5. On cycle detection or max depth, return null (caller synthesizes placeholder).

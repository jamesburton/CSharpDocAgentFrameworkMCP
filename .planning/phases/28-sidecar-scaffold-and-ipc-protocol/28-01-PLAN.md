   - Use camelCase property names (C# side uses `PropertyNameCaseInsensitive = true`)
   - Use integer values for enums to match existing MessagePack/JSON serialization convention

   - `schemaVersion: "1.0"`
   - `projectName`: derived from tsconfig path (directory name)
   - `sourceFingerprint: "stub"`
   - `contentHash: null`
   - `createdAt`: current ISO timestamp
   - `nodes: []`, `edges: []`
   - `ingestionMetadata: null`, `solutionName: null`

Create `src/DocAgent.Core/SymbolGraphDiffer.cs` as a `public static class` with a single public entry point:

```csharp
public static SymbolDiff Diff(SymbolGraphSnapshot before, SymbolGraphSnapshot after)
```

**Algorithm structure:**

1. **Validate** — throw `ArgumentException` if `before.ProjectName != after.ProjectName` (locked decision: error on incompatible snapshots).

2. **Index snapshots** — build `Dictionary<string, SymbolNode>` from each snapshot keyed by `SymbolId.Value`. Build edge indexes: `Dictionary<string, List<SymbolEdge>>` keyed by source `From.Value` for each snapshot.

3. **Find added symbols** — keys in afterIndex not in beforeIndex. For each, create a `SymbolChange` with `ChangeType.Added`, all detail fields null except a Description like "Symbol added". Determine ParentSymbolId by finding Contains edges pointing to this symbol in afterEdges.

4. **Find removed symbols** — keys in beforeIndex not in afterIndex. Same pattern as added but `ChangeType.Removed`, look up parent from beforeEdges.

5. **Find modified symbols** — keys in both indexes. For each pair (beforeNode, afterNode), call comparison methods for each category:

   a. **Signature changes** — compare `ReturnType`, `Parameters` (by position). Use `ParameterChange` entries for each parameter diff. Only emit if something actually changed.

   b. **Nullability changes** — compare `ReturnType` and each parameter's `TypeName` for nullable annotation differences (presence/absence of `?` suffix). Only emit if nullability specifically changed (not a full type change).

   c. **Constraint changes** — compare `GenericConstraints` lists. Match by `TypeParameterName`. Emit `ConstraintChangeDetail` with added/removed constraints per type parameter.

   d. **Accessibility changes** — compare `Accessibility` enum values directly. Emit `AccessibilityChangeDetail` with old/new values.

   e. **Dependency changes** — compare edges originating from this symbol. Group edges by `(From.Value, To.Value)` tuple. For same (From, To) pairs with different Kind, report as modification. For edges only in before, report as removed. For edges only in after, report as added. Emit one `DependencyChangeDetail` per symbol if any edge changes exist.

   f. **Doc comment changes** — compare `Docs` fields. If both null, no change. If one null and other not, or if `Summary`, `Remarks`, `Returns`, `Params`, or other fields differ, emit `DocCommentChangeDetail` with old/new `DocComment` objects.

6. **Severity determination** — per locked decision:
   - `DocComment` category → `Informational`
   - Symbol is `Public`, `Protected`, or `ProtectedInternal` → `Breaking` (for non-additive changes)
   - Added symbols (even public) → `NonBreaking` (additive)
   - All other accessibilities → `NonBreaking`

7. **Sort for determinism** — sort all changes by `SymbolId.Value` (StringComparer.Ordinal), then by `Category` (enum int value). This ensures identical output regardless of dictionary enumeration order.

8. **Build summary** — count changes by ChangeType and Severity to populate `DiffSummary`.

9. **Return** `new SymbolDiff(before.SourceFingerprint, after.SourceFingerprint, before.ProjectName, summary, sortedChanges)`.

**Implementation notes:**
- Keep all helper methods `private static` — this is a stateless utility.
- Use LINQ `Except`/`Intersect` on key sets for clarity, but remember to sort results.
- For ParentSymbolId resolution: scan edges for `SymbolEdgeKind.Contains` where `To == symbolId`.
- For nullability detection: a simple heuristic — if the type string differs only by trailing `?`, it's a nullability change. If the entire type changed, it's a signature change (don't double-report).

Implementation details for `Analyze`:

1. **Build findings from SymbolDiff.Changes** — Group by `SymbolId`. For each group:
   - Map `ChangeSeverity.Breaking` → `ReviewSeverity.Breaking`
   - Map `ChangeSeverity.NonBreaking` → `ReviewSeverity.Warning` if involves unusual pattern (see step 3), else `ReviewSeverity.Info`
   - Map `ChangeSeverity.Informational` → `ReviewSeverity.Info`
   - Build before/after strings from detail fields using switch on Category (see research code example for accessing detail fields)
   - Set `Category` to the ChangeCategory name
   - Set `Remediation` to actionable text based on category (e.g., Signature breaking → "Update all callers to match new signature", Accessibility widening → "Verify intent: widening accessibility increases public API surface")
   - `ImpactScope` is empty list at this level — populated by ChangeTools using snapshot edges

2. **Filter trivials when verbose=false** — Count and exclude `ChangeSeverity.Informational` + `ChangeCategory.DocComment` changes. Record count in `TrivialFiltered`.

3. **Detect unusual patterns** — scan all changes for:
   - **Accessibility widening**: `Category == Accessibility` && `AccessibilityDetail` old is more restrictive than new (Private < PrivateProtected < Internal < Protected < ProtectedInternal < Public). Use a static `AccessibilityRank` dictionary.
   - **Nullability regression**: `Category == Nullability` && `NullabilityDetail.NewAnnotation` contains "?" where `OldAnnotation` does not (for return types/params becoming nullable).
   - **Mass signature changes**: Group changes with `Category == Signature` by `ParentSymbolId`. Any group with count > `MassChangeThreshold` is flagged.
   - **Constraint removal**: `Category == Constraint` && `ConstraintDetail.RemovedConstraints.Count > 0`.

   When an unusual pattern is detected for a `NonBreaking` change, escalate that change's `ReviewSeverity` to `Warning` in the findings list.

4. **Compute summary**: Count findings per severity tier. `OverallRisk` = "high" if any Breaking, "medium" if any Warning, "low" otherwise.

5. **Sort findings**: Breaking first, then Warning, then Info. Within same severity, sort by SymbolId.

6. Return `ChangeReviewReport` with computed summary, filtered findings, and unusual findings list.

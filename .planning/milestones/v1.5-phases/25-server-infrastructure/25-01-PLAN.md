    **Step 1 — Create the Validation directory and StartupValidator.cs.**

    Create `src/DocAgent.McpServer/Validation/StartupValidator.cs` with these components:

    **ValidationResult record:**
    ```csharp
    public sealed record ValidationResult(
        IReadOnlyList<string> Errors,
        IReadOnlyList<string> Warnings)
    {
        public bool IsValid => Errors.Count == 0;
    }
    ```

    **Static Validate method** — pure function, no DI, no IO side effects except the ArtifactsDir writability probe:
    ```csharp
    public static ValidationResult Validate(DocAgentServerOptions options)
    ```

    Validation checks (in order):
    1. **AllowedPaths check:** If `options.AllowedPaths` is empty AND `Environment.GetEnvironmentVariable("DOCAGENT_ALLOWED_PATHS")` is null/whitespace, add a WARNING: `"AllowedPaths is empty and DOCAGENT_ALLOWED_PATHS is not set. PathAllowlist will default to cwd only."` This is a warning, not an error — the PathAllowlist has a safe default.
    2. **ArtifactsDir null/empty check:** If `options.ArtifactsDir` is null or whitespace, add an ERROR: `"ArtifactsDir is not configured. Set DocAgent:ArtifactsDir in appsettings.json or DOCAGENT_ARTIFACTS_DIR environment variable."`
    3. **ArtifactsDir writable check** (only if ArtifactsDir is non-null): Attempt to create the directory if it doesn't exist, then write a temp file (`Path.Combine(artifactsDir, ".startup-probe")`) and delete it. If any IOException occurs, add an ERROR: `"ArtifactsDir '{artifactsDir}' is not writable: {ex.Message}"`. Use a try/catch around the probe — do not let exceptions escape.

    **IHostedLifecycleService implementation:**
    - Namespace: `DocAgent.McpServer.Validation`
    - Class: `public sealed class StartupValidator : IHostedLifecycleService`
    - Constructor takes `IOptions<DocAgentServerOptions> options` and `ILogger<StartupValidator> logger` and `IHostApplicationLifetime lifetime`
    - Implement `StartingAsync` (the only meaningful method):
      1. Call `Validate(options.Value)`
      2. Log each warning at `LogLevel.Warning`
      3. Log each error at `LogLevel.Error`
      4. If `!result.IsValid`, log a final `LogLevel.Critical` message: `"Startup validation failed. Server will not accept tool calls."` then call `lifetime.StopApplication()` (this triggers graceful shutdown with non-zero exit)
    - All other lifecycle methods (`StartAsync`, `StopAsync`, `StartedAsync`, `StoppingAsync`, `StoppedAsync`) are no-ops returning `Task.CompletedTask`

    **Why `IHostedLifecycleService.StartingAsync` and not `IStartupFilter` or `IHostedService.StartAsync`:**
    - `StartingAsync` runs BEFORE `StartAsync` of all hosted services, ensuring validation happens before the MCP server transport starts accepting connections
    - This is the earliest reliable hook in the .NET hosting lifecycle

    **Why `lifetime.StopApplication()` instead of `Environment.Exit(1)`:**
    - `StopApplication()` triggers graceful shutdown, allowing the host to clean up resources
    - The host's exit code defaults to non-zero when `StopApplication()` is called from `StartingAsync` before the host finishes starting
    - If testing shows exit code is 0, add `Environment.ExitCode = 1` before calling `StopApplication()`

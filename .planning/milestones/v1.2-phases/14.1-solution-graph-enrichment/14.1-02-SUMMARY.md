# Phase 14.1 Plan 02: Stub Node Filtering in Search Indexes Summary

**NodeKind.Stub nodes are now excluded at index construction time from both BM25SearchIndex and InMemorySearchIndex, preventing external NuGet/framework stubs from polluting search rankings**

## Performance

- **Duration:** ~8 min
- **Started:** 2026-03-01
- **Completed:** 2026-03-01
- **Tasks:** 2
- **Files modified:** 3 (2 modified, 1 created)

## Accomplishments

- Added `Where(n => n.NodeKind == NodeKind.Real)` filter in `BM25SearchIndex.WriteDocuments` (Lucene document insertion) and `BM25SearchIndex.PopulateNodes` (_nodes dict for GetAsync)
- Added same filter in `InMemorySearchIndex.IndexAsync` — single filter covers both `SearchAsync` (iterates `_nodes.Values`) and `GetAsync` (looks up `_nodes`)
- Created `SolutionGraphEnrichmentTests` with 4 tests covering all 4 exclusion scenarios across both index implementations
- All 277 tests pass (273 pre-existing + 4 new)

## Task Commits

1. **Task 1: Stub filter in both index implementations** — `da3b7ce` (feat)
2. **Task 2: 4 stub filtering tests** — `f0693fb` (test)

## Files Created/Modified

- `src/DocAgent.Indexing/BM25SearchIndex.cs` — Filter in `WriteDocuments` and `PopulateNodes`; added `using System.Linq`
- `src/DocAgent.Indexing/InMemorySearchIndex.cs` — Filter in `IndexAsync`; added `using System.Linq`
- `tests/DocAgent.Tests/SolutionGraphEnrichmentTests.cs` — 4 tests: BM25 search exclusion, BM25 GetAsync exclusion, InMemory search exclusion, InMemory GetAsync exclusion

## Decisions Made

- Filter applied at two sites in BM25: `WriteDocuments` (Lucene docs) and `PopulateNodes` (_nodes dict) — both `SearchAsync` and `GetAsync` are covered
- Edges are NOT filtered — cross-project and external edges remain valid for reference graph traversal; only the node search indexes exclude stubs
- `using System.Linq` added explicitly to both files (project uses .NET 10, global using may not cover all implicit usings)

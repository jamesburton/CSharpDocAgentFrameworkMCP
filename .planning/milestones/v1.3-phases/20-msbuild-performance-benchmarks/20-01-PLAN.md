   - `[JsonExporter]` attribute on the class (produces JSON artifact for review)
   - `[GlobalSetup]` method that:
     - Resolves `_slnPath` to the repo's own `src/DocAgentFramework.sln` using `Path.GetFullPath` relative to the benchmark assembly location
     - Creates a temp directory for snapshot storage
     - Instantiates `SnapshotStore` with the temp directory
     - Instantiates `InMemorySearchIndex` (from DocAgent.Indexing)
     - Instantiates `SolutionIngestionService` with the store, search index, and `NullLogger`
     - Opens MSBuildWorkspace and caches it for warm benchmarks
     - Checks `workspace.Diagnostics` and logs warnings if workspace opened 0 projects
   - `[GlobalCleanup]` method that deletes the temp snapshot directory
   - Benchmark methods:
     a. `[Benchmark(Description = "FullSolutionIngestion")]` — calls `_service.IngestAsync(_slnPath, null, CancellationToken.None, forceFullReingest: true)`. This measures full ingestion latency (PERF-01) and memory (PERF-02).
     b. `[Benchmark(Description = "IncrementalNoChange")]` — calls `IngestAsync` a second time without changes, measuring incremental skip path latency.
     c. `[Benchmark(Description = "ColdWorkspaceOpen")]` — opens `MSBuildWorkspace.Create()`, loads solution, measures workspace open latency. Uses `[IterationSetup]`/`[IterationCleanup]` to create/dispose workspace fresh each iteration.

   **Important:** Use `.GetAwaiter().GetResult()` in `[GlobalSetup]` for async operations (InProcess toolchain pitfall). For `[Benchmark]` methods returning `Task`, BDN handles async correctly with the default out-of-process toolchain.

   **Note on workspace open:** The SolutionIngestionService internally opens MSBuildWorkspace. For the ColdWorkspaceOpen benchmark, directly use `MSBuildWorkspace.Create()` and `workspace.OpenSolutionAsync()` — this measures raw MSBuild latency without the ingestion pipeline overhead.

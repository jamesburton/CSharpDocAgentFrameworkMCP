   - `<PackageReference Include="BenchmarkDotNet" />` to DocAgent.Tests.csproj
   - `<ProjectReference Include="../../tests/DocAgent.Benchmarks/DocAgent.Benchmarks.csproj" />` to DocAgent.Tests.csproj

2. Create `tests/DocAgent.Tests/Performance/BaselineModels.cs`:
   - `BaselineFile` record with properties: `CapturedAt` (string), `Thresholds` (ThresholdConfig), `Baselines` (List of BaselineEntry)
   - `ThresholdConfig` record with `TolerancePercent` (double)
   - `BaselineEntry` record with: `BenchmarkName` (string), `MeanNanoseconds` (double), `AllocatedBytes` (long), `AbsoluteCeilingNanoseconds` (double?), `AbsoluteCeilingAllocatedBytes` (long?)
   - Use `System.Text.Json.Serialization.JsonPropertyNameAttribute` with camelCase names matching the JSON schema
   - All in namespace `DocAgent.Tests.Performance`

3. Create `tests/DocAgent.Tests/Performance/RegressionGuardTests.cs`:
   - Class `RegressionGuardTests` with `[Trait("Category", "Benchmark")]`
   - Single `[Fact]` method: `SolutionIngestion_DoesNotRegressBeyondBaseline`
   - Implementation:
     a. Compute path to baselines.json: `Path.GetFullPath(Path.Combine(AppContext.BaseDirectory, "../../../../../tests/DocAgent.Benchmarks/baselines.json"))`. If file not found, skip test with `Skip = "baselines.json not found"` or `Assert.Skip()`.
     b. Deserialize baselines.json into `BaselineFile` using `System.Text.Json.JsonSerializer`
     c. Configure BenchmarkDotNet:
        ```csharp
        var config = ManualConfig.Create(DefaultConfig.Instance)
            .WithOptions(ConfigOptions.DisableOptimizationsValidator)
            .AddDiagnoser(MemoryDiagnoser.Default)
            .AddJob(Job.ShortRun);  // 3 warmup, 5 target — fast for CI
        ```
     d. Run: `var summary = BenchmarkRunner.Run<SolutionIngestionBenchmarks>(config);`
     e. For each entry in `baselines.Baselines`:
        - Find matching report by `WorkloadMethodDisplayInfo == entry.BenchmarkName`
        - Assert report is not null
        - Compute `tolerance = 1.0 + (baselines.Thresholds.TolerancePercent / 100.0)`
        - **Latency check:** `stats.Mean <= entry.MeanNanoseconds * tolerance` with descriptive failure message
        - **Absolute latency ceiling:** if `entry.AbsoluteCeilingNanoseconds.HasValue`, assert `stats.Mean <= ceiling`
        - **Memory check:** `report.GcStats.GetBytesAllocatedPerOperation(report.BenchmarkCase) <= entry.AllocatedBytes * (long)tolerance` with descriptive failure message
        - **Absolute memory ceiling:** if `entry.AbsoluteCeilingAllocatedBytes.HasValue`, assert allocated <= ceiling

   **Key design decisions:**
   - Use `ConfigOptions.DisableOptimizationsValidator` because `dotnet test` runs in Debug by default. The 20% tolerance absorbs Debug vs Release measurement difference. Users who want precise measurement run the benchmark project directly in Release.
   - Use `Job.ShortRun` (3 warmup, 5 target iterations) for speed. Full benchmarks with BDN default iterations should be run via the benchmark project directly.
   - Use FluentAssertions for consistency with the rest of the test suite where natural, but standard xUnit Assert is fine for the threshold comparisons since the failure messages need custom formatting.
  </action>

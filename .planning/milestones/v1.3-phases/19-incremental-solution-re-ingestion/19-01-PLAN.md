**SolutionManifestStore** (in DocAgent.Ingestion namespace):

Create a static class `SolutionManifestStore` with these methods:

1. `ManifestFileName(string slnPath, string projectFilePath) -> string`: Computes a safe flat filename from the solution-relative path of the project file. Replace both `Path.DirectorySeparatorChar` and `Path.AltDirectorySeparatorChar` with `__`. Append `.manifest.json`. Example: `src/MyProject/MyProject.csproj` → `src__MyProject__MyProject.csproj.manifest.json`.

2. `ManifestDirectory(string artifactsDir) -> string`: Returns `{artifactsDir}/project-manifests`. Creates the directory if it doesn't exist.

3. `ComputeProjectManifestAsync(string projectFilePath, IReadOnlyList<string> projectReferencePaths, string? chosenTfm, CancellationToken ct) -> Task<FileHashManifest>`: Enumerates all `.cs` files in the project directory recursively (sorted by `StringComparer.Ordinal`), calls `FileHasher.ComputeManifestAsync`, then injects two synthetic hash entries:
   - Key `__project_refs__` = SHA-256 of sorted comma-joined project reference paths
   - Key `__tfm__` = SHA-256 of the TFM string (or empty string if null)
   Use `SHA256.HashData(Encoding.UTF8.GetBytes(...))` for the synthetic entries, same hex format as FileHasher.

4. `SaveAsync(string artifactsDir, string slnPath, string projectFilePath, FileHashManifest manifest, CancellationToken ct)`: Saves manifest to `ManifestDirectory/ManifestFileName` using `FileHasher.SaveAsync`.

5. `LoadAsync(string artifactsDir, string slnPath, string projectFilePath, CancellationToken ct) -> Task<FileHashManifest?>`: Loads manifest from `ManifestDirectory/ManifestFileName` using `FileHasher.LoadAsync`.

6. `CleanOrphanedManifests(string artifactsDir, IReadOnlySet<string> currentManifestFileNames)`: Deletes any `.manifest.json` files in `ManifestDirectory` whose filename is NOT in the provided set. This handles the "project moved" scenario.

**DependencyCascade** (in DocAgent.McpServer.Ingestion namespace):

Create an `internal static class DependencyCascade` with these methods:

1. `TopologicalSort(IReadOnlyList<ProjectEntry> projects, IReadOnlyList<ProjectEdge> edges) -> IReadOnlyList<string>`: Kahn's algorithm. Returns project names in dependency order (leaves first). Throws `InvalidOperationException` if cycle detected (should not happen if DetectCycles called first, but defensive).

2. `ComputeDirtySet(IReadOnlyList<string> directlyChanged, IReadOnlyList<ProjectEdge> edges) -> HashSet<string>`: BFS propagation from directly-changed projects through their dependents. Uses `StringComparer.OrdinalIgnoreCase`.

3. `HasStructuralChange(IReadOnlyList<ProjectEntry>? previousProjects, IReadOnlyList<string> currentProjectPaths) -> bool`: Returns true if project set differs (added or removed projects). Uses `OrdinalIgnoreCase` path comparison.

4. Extract `DetectCycles` from `SolutionIngestionService` (currently a private method) into this class as `internal static`. Update `SolutionIngestionService` to call `DependencyCascade.DetectCycles` instead. The existing tests must still pass.

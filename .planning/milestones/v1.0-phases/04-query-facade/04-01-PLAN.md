**Create `src/DocAgent.Indexing/KnowledgeQueryService.cs`:**

Constructor takes `ISearchIndex index`, `SnapshotStore snapshotStore`. Implements `IKnowledgeQueryService`.

**SearchAsync implementation:**
1. Start a `Stopwatch` for QueryDuration
2. Get the target snapshot: if `snapshotVersion` is null, get latest from `snapshotStore.ListAsync()` (most recent by CreatedAt); otherwise use the specified version
3. Load the snapshot via `snapshotStore.LoadAsync(contentHash)` — if null, return `QueryResult.Fail(SnapshotMissing)`
4. Determine staleness: compare the snapshot's `ContentHash` to the latest manifest entry's `ContentHash`
5. Call `_index.SearchAsync(query, ct)` to get raw hits
6. For each hit: look up the `SymbolNode` via `_index.GetAsync(hit.Id, ct)` — if `kindFilter` is set and node's Kind doesn't match, skip
7. Apply offset (skip first N filtered results) then collect up to `limit` results
8. Map each hit to `SearchResultItem(Id, Score, Snippet, node.Kind, node.DisplayName)`
9. Wrap in `ResponseEnvelope` with snapshot ContentHash, `DateTimeOffset.UtcNow`, staleness flag, elapsed duration
10. Return `QueryResult.Ok(envelope)`

**GetSymbolAsync implementation:**
1. Start Stopwatch
2. Resolve snapshot (same pattern: latest or pinned version)
3. Look up node via `_index.GetAsync(id, ct)` — if null, return `QueryResult.Fail(NotFound)`
4. Build navigation hints from snapshot.Edges:
   - `parentId`: first edge where `e.To == id && e.Kind == Contains` → `e.From`
   - `childIds`: edges where `e.From == id && e.Kind == Contains` → `e.To` list
   - `relatedIds`: edges where `(e.From == id || e.To == id) && e.Kind != Contains` → the other end, distinct
5. Create `SymbolDetail(node, parentId, childIds, relatedIds)`
6. Wrap in ResponseEnvelope, return Ok

**GetReferencesAsync stub:** `yield break;` (Phase 5/6 concern per MCPS-03).

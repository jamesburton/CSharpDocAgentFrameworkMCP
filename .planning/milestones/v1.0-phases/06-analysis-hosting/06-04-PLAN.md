<task id="2" name="Implement forceReindex support (MISSING-02)">

**Files:**
- `src/DocAgent.Core/Abstractions.cs` — add optional `forceReindex` parameter to `ISearchIndex.IndexAsync`
- `src/DocAgent.Indexing/BM25SearchIndex.cs` — honour `forceReindex` to bypass `IsIndexFresh` check
- `src/DocAgent.McpServer/Ingestion/IngestionService.cs` — pass `forceReindex` through to `_index.IndexAsync`

### Current state (important — understand before editing)

`IngestionService.IngestAsync` (lines 119-128) calls `_index.IndexAsync(saved, ct)` **unconditionally** in Stage 4. There is NO freshness check in IngestionService itself. The freshness check lives inside `BM25SearchIndex.IndexAsync` (line 69): `IsIndexFresh(dir, snapshot.ContentHash!)` — when fresh, it skips the rebuild and only populates `_nodes`.

The `forceReindex` parameter (line 48) is accepted by IngestionService but never read — it's a no-op.

### Fix approach

**Step 1:** Add `forceReindex` parameter to `ISearchIndex.IndexAsync`:

```csharp
// In Abstractions.cs:
Task IndexAsync(SymbolGraphSnapshot snapshot, CancellationToken ct, bool forceReindex = false);
```

The default value `false` keeps all existing callers compiling without changes.

**Step 2:** In `BM25SearchIndex.IndexAsync`, honour the parameter:

```csharp
public Task IndexAsync(SymbolGraphSnapshot snapshot, CancellationToken ct, bool forceReindex = false)
{
    // ... existing _injectedDirectory check ...

    // FSDirectory mode
    var indexPath = GetIndexPath(snapshot.ContentHash!);
    System.IO.Directory.CreateDirectory(indexPath);
    var dir = FSDirectory.Open(new System.IO.DirectoryInfo(indexPath));

    // Freshness check: skip rebuild if index already contains this hash (unless forced)
    if (!forceReindex && IsIndexFresh(dir, snapshot.ContentHash!))
    {
        PopulateNodes(snapshot);
        SwapActiveFsDirectory(dir);
        _hasIndex = true;
        return Task.CompletedTask;
    }
    // ... rest of rebuild logic unchanged ...
}
```

**Step 3:** In `IngestionService.IngestAsync`, pass `forceReindex` through (line 122):

```csharp
await _index.IndexAsync(saved, ct, forceReindex).ConfigureAwait(false);
```

**Test:** Add a test that calls `IngestAsync` twice with the same project — second call with `forceReindex=true` should trigger a full index rebuild (verify via mock or by checking the index is rebuilt). Second call with `forceReindex=false` should skip rebuild (freshness check passes).

</task>

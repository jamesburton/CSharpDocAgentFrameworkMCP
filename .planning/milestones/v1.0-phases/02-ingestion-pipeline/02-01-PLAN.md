1. Add `Microsoft.CodeAnalysis.Workspaces.MSBuild` version 4.12.0 to root `Directory.Packages.props`.
2. Add PackageReference for `Microsoft.CodeAnalysis.Workspaces.MSBuild` and `Microsoft.CodeAnalysis.CSharp.Workspaces` to `DocAgent.Ingestion.csproj`.
3. Create `LocalProjectSource.cs` implementing `IProjectSource`:

**DiscoverAsync logic:**
- If `locator.PathOrUrl` ends with `.csproj`, treat as explicit project path(s). Split on `;` for multiple paths.
- If it ends with `.sln`, use `MSBuildWorkspace.Create()` to open the solution via `OpenSolutionAsync`.
- If it's a directory, scan for `.sln` files. If exactly one found, open it. If multiple, throw with list. If none, scan for `.csproj` files.
- After loading, check `workspace.Diagnostics` and log warnings.

**Project filtering (per user decisions):**
- Exclude projects matching `*.Tests`, `*.Test`, `*.Specs` patterns (configurable via optional `includeTestProjects` parameter or similar).
- For multi-targeted projects, select highest TFM by comparing `project.ParseOptions` or the `TargetFramework` MSBuild property.

**Return `ProjectInventory`:**
- `RootPath`: directory containing the .sln or the common parent of csproj files.
- `SolutionFiles`: list of .sln paths found/used.
- `ProjectFiles`: filtered list of .csproj paths.
- `XmlDocFiles`: scan each project's output directory for matching `.xml` files (convention: `{AssemblyName}.xml` in output path).

**Important:** Roslyn 4.12+ uses out-of-process MSBuild — do NOT use MSBuildLocator. Just call `MSBuildWorkspace.Create()` directly.

**Compilation error handling (Claude's discretion):** Warn and continue — log compilation errors but proceed with available symbols. Skip only if Compilation is null.

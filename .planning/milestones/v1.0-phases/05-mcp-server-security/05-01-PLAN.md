**TRON serializer:** Create a new file `src/DocAgent.McpServer/Serialization/TronSerializer.cs`. TRON (Token Reduced Object Notation) defines schemas to avoid repeating property names. No .NET library exists, but the 5 response shapes are fixed and hand-rollable.

```csharp
namespace DocAgent.McpServer.Serialization;

/// <summary>
/// Minimal hand-rolled TRON serializer for the 5 fixed MCP response shapes.
/// TRON = Token Reduced Object Notation — a JSON superset that defines schemas
/// to reduce property name repetition, ideal for LLM token efficiency.
/// See https://tron-format.github.io/
/// </summary>
public static class TronSerializer
{
    /// <summary>Serialize search results: schema defines [id, score, kind, displayName, snippet] tuple.</summary>
    public static string SerializeSearchResults(IReadOnlyList<SearchResultItem> items);

    /// <summary>Serialize symbol detail with schema header.</summary>
    public static string SerializeSymbolDetail(SymbolDetail detail);

    /// <summary>Serialize reference edges: schema defines [sourceId, targetId, edgeKind, filePath, line] tuple.</summary>
    public static string SerializeReferences(IReadOnlyList<SymbolEdge> edges);

    /// <summary>Serialize diff entries: schema defines [id, changeKind, summary] tuple.</summary>
    public static string SerializeDiff(GraphDiff diff);

    /// <summary>Serialize project overview with nested schemas.</summary>
    public static string SerializeProjectOverview(object overview);
}
```

Each method emits a TRON document with a `$schema` header listing the ordered property names, followed by arrays of positional values. Example output for search results:
```json
{"$schema":["id","score","kind","displayName","snippet"],"data":[["N:MyApp",0.95,"Namespace","MyApp","Root namespace"],["T:MyApp.Foo",0.8,"Class","Foo","A class"]]}
```

Implement all 5 methods. Use `System.Text.Json.Utf8JsonWriter` for performance. The schemas are static — hardcode them per response shape.

**Error response helper:** Create a private `ErrorResponse(QueryErrorKind kind, string message, bool verbose)` method returning JSON with `error` code string (map `QueryErrorKind` to `path_denied`, `snapshot_not_found`, `invalid_symbol_id`, `invalid_input`, `stale_index`) and `message`. If `!verbose`, use opaque message for `path_denied`.

Create `StdoutContaminationTests.cs` with a single integration test class.

**Test approach:** Launch `dotnet run --project src/DocAgent.McpServer` as a subprocess. Pipe stdin to send a JSON-RPC `initialize` request followed by a `tools/list` request. Capture ALL stdout bytes. Assert:
1. Every line on stdout is valid JSON (parseable by `JsonDocument.Parse`)
2. No line contains common log prefixes: `info:`, `dbug:`, `warn:`, `fail:`, `trce:`
3. No line contains stack trace patterns: `at `, `Exception`, `Stacktrace`

**Implementation:**
```csharp
[Trait("Category", "Integration")]
public class StdoutContaminationTests : IAsyncLifetime
{
    private Process? _process;

    public async Task InitializeAsync()
    {
        // Build first to ensure latest code
        // (skip if CI handles this)
    }

    public Task DisposeAsync()
    {
        _process?.Kill();
        _process?.Dispose();
        return Task.CompletedTask;
    }

    [Fact]
    public async Task Stdout_ContainsOnlyJsonRpc_NoLogContamination()
    {
        var startInfo = new ProcessStartInfo
        {
            FileName = "dotnet",
            Arguments = "run --project src/DocAgent.McpServer",
            RedirectStandardInput = true,
            RedirectStandardOutput = true,
            RedirectStandardError = true,
            UseShellExecute = false,
            WorkingDirectory = GetRepoRoot(), // navigate to repo root
        };

        _process = Process.Start(startInfo)!;

        // Send JSON-RPC initialize
        var initRequest = JsonSerializer.Serialize(new
        {
            jsonrpc = "2.0",
            id = 1,
            method = "initialize",
            @params = new { protocolVersion = "2024-11-05", capabilities = new { }, clientInfo = new { name = "test", version = "1.0" } }
        });

        // MCP uses content-length framing — write with appropriate framing
        // Note: The MCP stdio transport may use newline-delimited JSON or Content-Length headers.
        // Test must match the transport framing used by the SDK.
        // Try newline-delimited first; adjust if SDK uses Content-Length.
        await _process.StandardInput.WriteLineAsync(initRequest);
        await _process.StandardInput.FlushAsync();

        // Read response with timeout
        var cts = new CancellationTokenSource(TimeSpan.FromSeconds(30));
        var stdoutLines = new List<string>();

        // Read available stdout for a few seconds
        var readTask = Task.Run(async () =>
        {
            while (!cts.Token.IsCancellationRequested)
            {
                var line = await _process.StandardOutput.ReadLineAsync();
                if (line is null) break;
                stdoutLines.Add(line);
                if (stdoutLines.Count >= 5) break; // enough to verify
            }
        });

        await Task.WhenAny(readTask, Task.Delay(10_000, cts.Token));
        cts.Cancel();

        // Assert: every non-empty stdout line is valid JSON
        foreach (var line in stdoutLines.Where(l => !string.IsNullOrWhiteSpace(l)))
        {
            var parseAction = () => JsonDocument.Parse(line);
            parseAction.Should().NotThrow($"stdout line should be valid JSON: '{line.Substring(0, Math.Min(100, line.Length))}'");
        }

        // Assert: no log contamination patterns
        var allStdout = string.Join("\n", stdoutLines);
        allStdout.Should().NotContainAny("info:", "dbug:", "warn:", "fail:", "trce:");
        allStdout.Should().NotContain("Exception");
    }

    private static string GetRepoRoot()
    {
        var dir = Directory.GetCurrentDirectory();
        while (dir is not null && !File.Exists(Path.Combine(dir, "DocAgentFramework.sln"))
                                && !Directory.Exists(Path.Combine(dir, "src")))
            dir = Directory.GetParent(dir)?.FullName;
        return dir ?? throw new InvalidOperationException("Cannot find repo root");
    }
}
```

**Important notes:**
- Mark test with `[Trait("Category", "Integration")]` so it can be filtered in CI
- The exact MCP stdio framing may need adjustment — if the SDK uses Content-Length headers instead of newline-delimited JSON, adapt the write/read to match. Check the research: the SDK handles framing via `WithStdioServerTransport()`.
- Use a generous timeout (30s) — `dotnet run` needs to compile if not cached.
- **Known scope boundary:** `IKnowledgeQueryService` has no concrete DI registration in Program.cs yet (it's a TODO for full pipeline assembly). The subprocess will fail at DI resolution when a tool is actually invoked. This test only sends `initialize` + `tools/list` (which don't invoke tool handlers), so DI resolution of `IKnowledgeQueryService` is NOT triggered. The test validates stdout purity of the host startup and MCP handshake, not tool execution.
- If the process fails to start for any reason, the test should still verify stdout purity for whatever output is produced (even error frames should be valid JSON-RPC).

1. **TypeScriptIngestionException.cs** — Per CONTEXT.md:
   - `public sealed class TypeScriptIngestionException : Exception`
   - Properties: `int ExitCode`, `string StderrOutput`
   - Constructor: `(int exitCode, string stderrOutput)` with base message `$"TypeScript sidecar exited with code {exitCode}: {stderrOutput}"`

2. **TypeScriptIngestionService.cs** — Following IngestionService pattern:
   - Constructor: inject `IOptions<DocAgentServerOptions>`, `ILogger<TypeScriptIngestionService>`, `SnapshotStore`, `ISearchIndex`
   - `internal Func<string, Task<SymbolGraphSnapshot>>? PipelineOverride { get; set; }` — test seam per CONTEXT.md
   - `public async Task<IngestionResult> IngestTypeScriptAsync(string tsconfigPath, CancellationToken ct)`
   - If PipelineOverride is set, call it and return result
   - Otherwise: spawn `node {sidecarDir}/dist/index.js` via `System.Diagnostics.Process`
     - Set `UseShellExecute = false`, redirect stdin/stdout/stderr, `CreateNoWindow = true`
     - `WorkingDirectory` = sidecar directory
     - Call `BeginErrorReadLine()` BEFORE reading stdout (avoid deadlock per RESEARCH.md pitfall 2)
     - Capture stderr lines asynchronously, forward each to `ILogger.LogDebug("[ts-sidecar] {Line}", line)`
     - Write JSON-RPC request to stdin: `{"jsonrpc":"2.0","id":1,"method":"extract","params":{"tsconfigPath":"..."}}`
     - Close stdin after writing (signal EOF)
     - Read single response line from stdout
     - Wait for exit with timeout from `_options.IngestionTimeoutSeconds` (default 300s)
     - If exit code != 0, throw `TypeScriptIngestionException(exitCode, stderrContent)`
     - Deserialize response using `JsonSerializer.Deserialize<JsonElement>` first, extract `result` property, then deserialize to `SymbolGraphSnapshot`
     - Use `PropertyNameCaseInsensitive = true` in JsonSerializerOptions (per RESEARCH.md)
     - Do NOT add `JsonStringEnumConverter` — use integer enum values (per RESEARCH.md recommendation)
   - After getting snapshot: save to SnapshotStore, index in SearchIndex (same pattern as IngestionService)
   - Per-path SemaphoreSlim serialization (same pattern as IngestionService)
   - Sidecar directory: resolve from `_options.SidecarDir` or default to `src/ts-symbol-extractor` relative to working directory

3. **DocAgentServerOptions.cs** — Add one property:
   - `public string? SidecarDir { get; set; }` — path to ts-symbol-extractor directory. Default null (auto-detect).

4. **ServiceCollectionExtensions.cs** — Add registration:
   - `services.AddSingleton<TypeScriptIngestionService>();` after existing IngestionService registration

   - Verify IngestTypeScriptAsync returns the override result
   - Verify exception propagation
   - Verify null path validation
   - Verify TypeScriptIngestionException properties

1. **NodeAvailabilityValidator.cs** — Following StartupValidator pattern:
   - `public sealed class NodeAvailabilityValidator : IHostedLifecycleService`
   - Constructor: inject `ILogger<NodeAvailabilityValidator>`, `IOptions<DocAgentServerOptions>`
   - Make validation logic testable via static/internal methods:
     - `internal static NodeCheckResult ParseNodeVersion(string? versionOutput)` — strips "v" prefix, uses `Version.TryParse`, checks >= 22.0
     - `internal static SidecarBuildResult CheckSidecarBuild(string sidecarDir)` — checks if `dist/index.js` exists
   - `NodeCheckResult` record: `(bool IsAvailable, bool IsSupported, Version? ParsedVersion)`
   - `SidecarBuildResult` record: `(bool NeedsBuild, string DistPath)`
   - `StartingAsync` implementation:
     1. Run `node --version` via Process, capture stdout
     2. Parse output with `ParseNodeVersion`
     3. If not available: `logger.LogWarning("Node.js is not installed. TypeScript ingestion will not be available.")`
     4. If available but not supported: `logger.LogWarning("Node.js {Version} found but >= 22.x is required. TypeScript ingestion may not work correctly.", version)`
     5. If available and supported: `logger.LogInformation("Node.js {Version} detected.", version)`
     6. Resolve sidecar directory from options (same logic as TypeScriptIngestionService)
     7. If sidecar dir exists, check `dist/index.js`:
        - If missing AND Node.js is available: run `npm install && npm run build` in sidecar dir
        - Log progress: `logger.LogInformation("Building TypeScript sidecar (first run)...")`
        - If build fails: log warning, don't crash
     8. CRITICAL: Do NOT call `_lifetime.StopApplication()` on failure — per CONTEXT.md, log warning only. C# ingestion still works without Node.js.
   - All other lifecycle methods: `return Task.CompletedTask;`

<task type="auto">
  <name>Task 1: Deterministic SymbolId Generation</name>
  <files>src/ts-symbol-extractor/src/symbol-id.ts</files>
  <action>
    Implement `getSymbolId` and related helpers in `symbol-id.ts`.
    The ID must be deterministic and follow the pattern: `[ProjectName]:[RelativePath]:[SymbolPath]`.
    - `ProjectName`: Root directory name containing the tsconfig.json.
    - `RelativePath`: Path from project root to the source file (e.g., `src/utils.ts`).
    - `SymbolPath`: Dot-separated path to the declaration (e.g., `MyClass.myMethod`).
    - Handle overloads by appending a suffix or parameter types to the `SymbolPath`.
    - Ensure IDs are collision-free within a project and stable across runs.
  </action>
  <verify>
    Create a temporary test file and verify that the same `ts.Symbol` always produces the same ID string.
  </verify>
  <done>
    `getSymbolId` correctly generates stable IDs for namespaces, types, and members.
  </done>
</task>

<task type="auto">
  <name>Task 2: Source File Walker and Namespace Mapping</name>
  <files>src/ts-symbol-extractor/src/extractor.ts, src/ts-symbol-extractor/src/index.ts</files>
  <action>
    Implement `extractSymbols(tsconfigPath: string): SymbolGraphSnapshot` in `extractor.ts`.
    1. Use `ts.createProgram` to load the project from `tsconfigPath`.
    2. Get the type checker: `program.getTypeChecker()`.
    3. Identify project files via `program.getRootFileNames()`.
    4. Walk each `SourceFile`:
       - Exclude files in `node_modules`.
       - Map each file to a `SymbolKind.Namespace` node.
       - Use the file's relative path for the namespace name.
       - Extract source spans (filePath, start/end line/column).
    5. Update `index.ts` to call the real `extractSymbols` instead of `extractStub`.
  </action>
  <verify>
    `npm run build` succeeds and `npm test` passes with basic walker tests.
  </verify>
  <done>
    Extractor correctly walks project source files and maps them to namespace nodes with correct spans.
  </done>
</task>

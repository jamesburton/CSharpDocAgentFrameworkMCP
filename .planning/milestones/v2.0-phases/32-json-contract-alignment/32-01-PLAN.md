Fix the JSON contract between TypeScript sidecar and C# deserialization so that sidecar output correctly deserializes into SymbolGraphSnapshot domain types.

</objective>

**Step 2: Create `DocCommentConverter` in `src/DocAgent.McpServer/Ingestion/DocCommentConverter.cs`.**

Create a custom `JsonConverter<DocComment>` that handles shape mismatches:
- TS `example: string|null` -> C# `Examples: IReadOnlyList<string>` (wrap single string in list)
- TS `throws: Record<string,string>` -> C# `Exceptions: IReadOnlyList<(string Type, string Description)>` (dict entries to tuple list)
- TS `see: string[]` -> C# `SeeAlso: IReadOnlyList<string>` (name mapping)
- TS `params: Record<string,string>` -> C# `Params: IReadOnlyDictionary<string,string>` (compatible with case insensitive)
- TS `typeParams: Record<string,string>` -> C# `TypeParams: IReadOnlyDictionary<string,string>` (compatible)
- `summary`, `remarks`, `returns` map directly (case insensitive handles it)

The `Write` method should `throw new NotSupportedException("DocCommentConverter is read-only (sidecar deserialization only)")` — MCP output uses a separate serializer.

Follow the exact pattern from the research (Pattern 3 in 32-RESEARCH.md). Use `JsonDocument.ParseValue(ref reader)` to read the raw JSON and construct the C# DocComment record.

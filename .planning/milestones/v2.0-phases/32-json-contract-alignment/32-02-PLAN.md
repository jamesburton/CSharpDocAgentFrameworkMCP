Create a test class `TypeScriptDeserializationTests` with `[Trait("Category", "Deserialization")]` on the class. The test class needs to construct the same `JsonSerializerOptions` used by `TypeScriptIngestionService` (since it is private). Create a static helper method that builds identical options:

```csharp
private static readonly JsonSerializerOptions SidecarJsonOptions = new()
{
    PropertyNameCaseInsensitive = true,
```

Tests to create:

1. **`GoldenFile_Deserializes_To_Valid_Snapshot`**: Load the golden JSON file, deserialize using `JsonSerializer.Deserialize<SymbolGraphSnapshot>(json, SidecarJsonOptions)`. Assert:
   - Snapshot is not null
   - `Nodes` is not empty
   - `Edges` is not empty
   - At least one `SymbolId.Value` is non-empty (verifies record struct deserialization)

2. **`GoldenFile_Edge_Integrity`**: Deserialize golden file. Assert:
   - ALL edges have non-null `From` and non-null `To` with non-empty `Value`
   - Contains edges exist (kind == SymbolEdgeKind.Contains)
   - If the fixture has class inheritance: an `Inherits` edge exists from SpecialGreeter to Greeter (or equivalent fixture classes)
   - If the fixture has interface implementation: an `Implements` edge exists
   - No edge has kind == 0 (would indicate numeric fallback)

3. **`GoldenFile_DocComment_Preservation`**: Deserialize golden file. Assert:
   - At least one node has `Docs != null`
   - That node's `Docs.Summary` is not null/empty and matches expected text (e.g., "This is a sample function." for the hello function)
   - `Docs.Params` is a valid dictionary (may be empty if no @param tags)

4. **`GoldenFile_Enum_String_Deserialization`**: Deserialize golden file. Assert:
   - All nodes have valid `Kind` values (not default/zero unless intentionally Namespace=0)
   - All nodes have valid `Accessibility` values
   - Specifically verify a Type node has `Kind == SymbolKind.Type` and a Method node has `Kind == SymbolKind.Method`

5. **`GoldenFile_SymbolNode_Docs_From_DocComment_Field`**: Specifically tests the `docComment` -> `Docs` JsonPropertyName mapping. Find a node known to have docs in the fixture and verify `Docs` is populated (not null).

6. **`GoldenFile_Snapshot_Matches_Reference`**: Full snapshot structural equality test (per locked assertion category 3). Deserialize golden file and assert:
   - `Nodes.Count` matches expected count (capture from golden file on first run, then hardcode)
   - `Edges.Count` matches expected count
   - At least 3 representative `SymbolId.Value` strings are present in the node set (e.g., the top-level namespace, a class, and a method — pick stable IDs from the fixture)
   - `SchemaVersion` is non-null/non-empty
   - `ProjectName` matches the fixture project name
   - All edge `From`/`To` SymbolIds exist as node IDs in the snapshot (referential integrity)
   This is NOT a byte-level comparison but a structural equality check that catches missing nodes, dropped edges, or broken ID references.

Use `FluentAssertions` for all assertions. Use `Should().Contain(...)` for structural assertions, not exact count assertions (per research anti-pattern guidance) — EXCEPT for `GoldenFile_Snapshot_Matches_Reference` where exact counts ARE appropriate since we control the golden file.

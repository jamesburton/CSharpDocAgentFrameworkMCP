## Cross-Phase Integration

**Score:** 17 connected / 1 orphaned export

### Connected (17 verified wiring paths)

**Phase 13 → 14:** Domain types consumed by SolutionIngestionService
- `SymbolNode.ProjectOrigin` stamped on all nodes during ingestion
- `SymbolGraphSnapshot.SolutionName` populated from .sln path
- `ISolutionIngestionService` registered in DI, injected into `IngestionTools`

**Phase 13 → 14.1:** Enrichment types populated
- `SolutionSnapshot`, `ProjectEntry`, `ProjectEdge` — constructed and populated in `IngestAsync`
- `EdgeScope.CrossProject` — classified at construction in `ClassifyScope`
- `NodeKind.Stub` — synthesized in `MaybeAddStubNode` with 30-entry primitive filter

**Phase 14.1 → 15:** Index and query layer
- Stub filtering: `BM25SearchIndex` + `InMemorySearchIndex` both filter `.Where(n.NodeKind == NodeKind.Real)`
- `projectFilter` applied in `KnowledgeQueryService.SearchAsync`
- `crossProjectOnly` filtering in `GetReferencesAsync` using `EdgeScope.CrossProject`

**Phase 15 → MCP surface:**
- `search_symbols` has `project` param → `projectFilter` passthrough
- `get_references` has `crossProjectOnly` param → passthrough + enriched JSON (scope, fromProject, toProject)
- `get_symbol` has FQN disambiguation via `ResolveByFqnAsync`
- `SearchResultItem.ProjectName` serialized in JSON output

### Orphaned Export (1)

**`SolutionIngestionResult.Snapshot`** — `SolutionSnapshot` populated in memory during `IngestAsync` but never persisted to `SnapshotStore`. No MCP tool or query method reads it. Planned Phase 16 gap — `explain_solution` and `diff_snapshots` tools will consume it.

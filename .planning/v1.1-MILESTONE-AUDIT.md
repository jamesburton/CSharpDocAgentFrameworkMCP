### Unsatisfied / Partial Requirements

**R-CHANGE-TOOLS (partial):** All 3 MCP tools are implemented and functionally correct (10/10 tests pass), but `ChangeTools` does not enforce the `PathAllowlist` security gate. The `_allowlist` field is injected via constructor but `IsAllowed()` is never called in any of the 3 tool methods (`ReviewChanges`, `FindBreakingChanges`, `ExplainChange`). This breaks the security contract established by all peer tool classes (`DocTools.cs` line 163, `IngestionTools.cs` line 67).

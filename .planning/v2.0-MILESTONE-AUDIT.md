### Connected (16)

- DocCommentConverter bridges all TS→C# shape mismatches (example→Examples, throws→Exceptions, see→SeeAlso)

### Integration Issues (4)

**INT-01 (Medium): GenericConstraint field name mismatch**
- TS emits `name` field, C# expects `typeParameterName` via `[JsonPropertyName("typeParameterName")]`
- Result: generic type constraint data silently dropped during deserialization
- Files: `src/ts-symbol-extractor/src/types.ts:127`, `src/DocAgent.Core/Symbols.cs:81`
- Affects: SIDE-03, EXTR-01

**INT-02 (Medium): Sidecar E2E tests skip silently in CI**
- `TypeScriptSidecarIntegrationTests` uses early-return guard, not `[Fact(Skip=...)]`
- Tests show as "passed" without exercising real sidecar in standard `dotnet test`
- Files: `tests/DocAgent.Tests/TypeScriptSidecarIntegrationTests.cs:85,145`
- Affects: SIDE-03, MCPI-01, MCPI-02

**INT-03 (Low): AppHost sidecar resource variable unused**
- `AddNodeApp()` return value not passed to McpServer via `WithReference()` or `WaitFor()`
- Aspire dashboard shows no dependency relationship between ts-sidecar and docagent-mcp
- Files: `src/DocAgent.AppHost/Program.cs:10`
- Affects: SIDE-04

**INT-04 (Low): TS edge kinds not in C# enum**
- TS `types.ts` defines `Accepts` and `InheritsFrom` in `SymbolEdgeKind` but C# enum lacks them
- Currently dormant — extractor never emits these. Latent throw risk if emitted in future.
- Files: `src/ts-symbol-extractor/src/types.ts:138-139`
- Affects: EXTR-04

## Cross-Phase Integration

### Critical Issue: JSON Deserialization Mismatch

The TypeScript sidecar output format does not match C# domain types. This is the **root cause** of 6 partial/unsatisfied requirements:

| Mismatch | TS Output | C# Expected | Impact |
|----------|-----------|-------------|--------|
| Edge properties | `sourceId` / `targetId` | `From` / `To` | All edges lost (null From/To) |
| Doc property | `docComment` | `Docs` | All documentation dropped |
| EdgeKind enum ordinals | `Inherits=6`, `Extends=1` | `Inherits=1`, no `Extends` | Edge types corrupted |

**Why not caught:** 100% of tests use `PipelineOverride` which injects in-memory C# objects, completely bypassing the JSON deserialization path.

**Fix approaches:**
1. Add `[JsonPropertyName]` attributes to C# records to match TS output
2. Add JSON naming policy to `TypeScriptIngestionService.JsonOptions`
3. Update TS `types.ts` to match C# names and fix enum ordinals
4. Add end-to-end integration test that exercises real sidecar → JSON → C# path

### Missing: Aspire AppHost Integration (SIDE-04)

`src/DocAgent.AppHost/Program.cs` does not register the Node.js sidecar. `NodeAvailabilityValidator` provides startup validation as a hosted service but is not an Aspire resource.
